Этот скилл ведёт персональную доску «На обдумывание» для одного радара проекта **Radar** —
персистентную память, к которой пользователь возвращается, чтобы обдумать находки.
Это **не** рейтинг: рейтинг (`ratings.json` + `learn-profile`) учит тематику; закладка
(`saved.json`) — это «хочу подумать об этом позже», с заметкой и статусом.

## Вход

Аргумент — `slug` радара (напр. `claude-code`). Пути относительно `radars/<slug>/`.
Файл закладок: `data/feedback/saved.json`. Страница: `THINKING.md` (генерируется).

## Модель данных (saved.json)

`items[]`, каждый:
- `find_id` — ссылка на `find.id` из `data/finds/*.json`.
- `saved_at` — дата добавления (YYYY-MM-DD).
- `status` — `new` | `thinking` | `done` | `archived`.
- `note` — личная заметка/мысль (markdown, может быть пусто).
- `note_updated_at` — дата правки заметки.

## Операции (по запросу пользователя)

**Отложить находку.** Найди `find_id` (по заголовку/номеру в дайджесте или прямому id).
Добавь item со `status: "new"`, `saved_at` = сегодня. Если уже отложена — не дублируй,
сообщи об этом. (Дату «сегодня» бери реальную; не выдумывай.)

**Добавить/изменить заметку.** Найди item по `find_id`, обнови `note` и `note_updated_at`.

**Сменить статус.** new → thinking → done → archived (или любой по просьбе).

**Показать доску.** Прочитай `saved.json`, покажи сгруппированно по темам и статусам.

**Удалить из доски.** Убери item (или переведи в `archived`, если пользователь хочет сохранить историю).

После любой записи обнови `updated_at` в файле.

## Пересборка страницы

После изменений всегда пересобирай человекочитаемую страницу:
```
python3 engine/build_thinking.py <slug>
```
Скрипт группирует по темам (первый тег находки) и статусам, подтягивает заголовки/ссылки
из `data/finds/*.json` и пишет `radars/<slug>/THINKING.md`. Это общий движок — не дублируй
его логику, просто вызывай.

## Замечание про сайт

На сайте (Этап 3) кнопка «⚑ Отложить» пишет в тот же `saved.json`, а заметки/статусы
редактируются в UI на странице «На обдумывание». Формат файла един — и чат, и сайт правят одно хранилище.

## Чего не делать

- Не путать с рейтингами: не трогать `ratings.json`.
- Не выдумывать `find_id` — только реально существующие в `data/finds/*.json`.
- Не дублировать уже отложенные находки.
