## 7. Полный гайд: хуки, субагенты и скиллы Claude Code — когда что применять

**Анонс.** Разбор трёх слоёв расширяемости Claude Code: хуки (детерминированное правило кодом), субагенты (изолированная параллельная работа в своём контексте) и скиллы (переиспользуемые инструкции, грузятся только при вызове). С чёткими правилами выбора: slash-команда / скилл / субагент / хук под конкретную задачу.

**Расшифровка.** Практическая навигация по расширяемости: хук — когда нужно правило, которое модель не может «забыть» (срабатывает кодом); субагент — для изолированной/параллельной работы, его подробный вывод не засоряет основной контекст, возвращается только итог; скилл — когда есть доменная логика или вспомогательные файлы, тело грузится лишь при вызове (дёшево по контексту). Правило выбора: slash-команда для шаблона промпта, скилл для доменной логики, субагент для изолированной параллельной работы, хук для детерминированного контроля. Старт рекомендуют со скиллов — проще всего и сразу полезно.

## 9. Практика: субагенты Claude Code в проде — как делегировать и не потерять контроль

**Анонс.** Прикладной гайд (Nimbalyst) по субагентам в реальной работе: субагент — это специалист со своим системным промптом, набором инструментов и правами; его многословный вывод (поиск файлов, дампы логов) остаётся изолированным, а в основную беседу возвращается только итог. Разбор, когда делегировать и как сохранять управляемость.

**Расшифровка.** Практический разбор «как это работает у нас в проде»: субагент изолирует контекст — его шумная работа (grep по файлам, длинные логи, многошаговое рассуждение) не засоряет главную сессию, наверх идёт только сводка. Это и экономит контекст, и повышает управляемость. Гайд даёт ориентиры: для какой работы заводить субагента (изолированная, параллельная), как давать ему узкие права инструментов, как не уйти в неуправляемое дерево делегирования. Хорошо ложится рядом с официальной фичей вложенных субагентов (до 5 уровней) из Week 24.
