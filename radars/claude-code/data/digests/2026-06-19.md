## 1. Claude Code v2.1.183 (19 июня): auto mode блокирует деструктивные git/IaC-команды, /config --help

**Анонс.** Релиз v2.1.183 от 19 июня: в auto mode теперь блокируются опасные git-команды (reset --hard, checkout -- ., clean -fd, stash drop), если ты не просил сбросить локальную работу; commit --amend — если коммит не сделан агентом в этой сессии; terraform/pulumi/cdk destroy — без явного указания стека. Добавлен /config --help, предупреждение об устаревшей модели в -p.

**Расшифровка.** Важное усиление безопасности автономного режима — прямо в тему наших автоматизаций. Auto mode перестаёт выполнять разрушительные операции без явного запроса: git reset --hard, git checkout -- ., git clean -fd, git stash drop (потеря несохранённого), git commit --amend на чужом коммите, а также terraform/pulumi/cdk destroy без указания конкретного стека. Плюс: предупреждение на stderr в print-режиме (-p), если запрошенная модель устарела/авто-обновлена (в т.ч. модель из frontmatter агента); attribution.sessionUrl — убрать ссылку на claude.ai-сессию из коммитов/PR; /config --help со списком ключей. Фиксы: мерцание в JetBrains-терминалах 2026.1+, висящая валидация PowerShell-команд на Windows, WebSearch возвращал пусто в субагентах.

## 2. Artifacts в Claude Code: сессия превращается в живую shareable веб-страницу (beta, Team/Enterprise)

**Анонс.** Anthropic (18 июня) добавила Artifacts в Claude Code: работа сессии превращается в живые shareable-страницы — PR-разборы, дашборды, system explainer'ы, релиз-чеклисты. Страница сама обновляется по тому же URL при изменениях, есть история версий. Beta для Team/Enterprise, из CLI и desktop, просмотр в любом браузере.

**Расшифровка.** Заметная фича: достаточно попросить в сессии «сделай artifact» (или что-то визуальное) — Claude Code собирает страницу из всего контекста сессии (репозитории, подключённые данные, история чата), без ручной загрузки. Возможности: live-обновление (открытая страница рефрешится сама, тиммейты видят изменения сразу), история версий (каждая публикация — новая версия по тому же URL, можно откатиться), галерея артефактов. Приватность: по умолчанию видно только участникам организации, публичный шеринг запрещён, админы управляют видимостью и ролями. Доступно в beta на Team/Enterprise из CLI и desktop-приложения.

`ide-integration · workflows · automation · tips` · Источник: [docs/Anthropic](https://claude.com/blog/artifacts-in-claude-code)

## 3. Enterprise-managed авторизация MCP-коннекторов: zero-touch доступ через Okta (включая Claude Code)

**Анонс.** Anthropic (18 июня) запустила в beta централизованную авторизацию MCP-коннекторов через IdP — старт с Okta. Админы провижинят коннекторы на всю организацию, сотрудники получают доступ автоматически при первом входе (zero-touch), по группам/ролям из IdP. Работает единообразно в Claude chat, Claude Code и Cowork. Партнёры: Asana, Atlassian, Canva, Figma, Granola, Linear, Supabase (Slack скоро).

**Расшифровка.** Корпоративное упрощение MCP, затрагивающее Claude Code: вместо per-connector OAuth каждому юзеру — админ подключает IdP (старт с Okta) и выбирает, какие MCP-коннекторы включить; доступ выдаётся по уже существующим группам/ролям. Цитата клиента: «2000 сотрудников, провижинятся через Okta, ноль доп. шагов — в первый день уже подключены». Отзыв доступа — через IdP. Это первая реализация расширения Enterprise-Managed Authorization к спеке MCP, открытого — любой коннектор, в т.ч. кастомный, может его принять. Beta для Team/Enterprise; единый доступ через Claude chat, Claude Code и Cowork.

`mcp · automation · ci-cd` · Источник: [docs/Anthropic](https://claude.com/blog/enterprise-managed-auth)

## 7. Кейс-гайд: оптимизация токенов Claude Code — как не получить счёт на $1600

**Анонс.** Практический разбор на реальных сессиях, на что утекают токены в Claude Code и как сократить расход: основной драйвер — не промпты, а накапливающийся контекст; токенайзер Opus 4.8 добавляет до 35% токенов на промпт. Рецепты: компактный CLAUDE.md, /clear и /compact при переключении задач, ограничение длины вывода терминала, мониторинг burn-rate.

**Расшифровка.** Полезно на фоне биллинг-истории: основной драйвер счёта — не сами промпты, а раздувающийся контекст за ними; плюс токенайзер Opus 4.8 добавляет до ~35% токенов на промпт, так что эффективная стоимость выше «ценника». Рецепты сокращения: держать CLAUDE.md компактным (5000-токенный файл стоит 5000 токенов до первого слова); /clear при смене задачи и /compact на длинных — суммаризирует и сбрасывает старые данные; ограничивать длину вывода bash (длинные тест-логи жрут токены); мониторить burn-rate в первую неделю цикла и right-size план. Прямо применимо к нашим ежедневным автоматизациям на claude -p.

## 8. Кейс-гайд: хуки, субагенты и power-фичи Claude Code на практике (slash-команды под команду)

**Анонс.** Свежий практический гайд (DEV/vibehackers): CLAUDE.md под проект (<200 строк), хуки для «непреложного» (что должно срабатывать на 100% — это хук, а не инструкция), субагенты под повторяющиеся задачи (ревью, тесты, доки — со своей моделью/инструментами), кастомные slash-команды под рутину команды (/deploy, /pr-summary, /release-notes).

**Расшифровка.** Прикладная сборка лучших практик «как это применить у себя»: 1) CLAUDE.md — старт проекта: команды сборки, заметки по архитектуре, стандарты кода, держать <200 строк; 2) хуки — для непреложного: всё, что должно происходить всегда, оформляй хуком, а не текстовой инструкцией (модель может «забыть», хук — нет); 3) субагенты — на повторяющуюся работу (code review, написание тестов, документация) с правильной моделью, инструментами и инструкциями; 4) кастомные slash-команды под то, что команда делает регулярно (/deploy, /pr-summary, /release-notes). Три слоя расширяемости (хуки/субагенты/скиллы) и когда что комбинировать — с примерами под реальный командный воркфлоу.
