## 1. Subagents vs Agent Teams vs Workflows: какой примитив параллелизма выбрать

**Анонс.** Developers Digest сравнивает три способа Claude Code распараллелить задачу по тому, кто управляет оркестрацией: субагенты/скиллы (решает Claude пошагово), agent teams (воркеры как равные, общий список задач и обмен сообщениями) и dynamic workflows (оркестрация вынесена в JS-скрипт). Вывод: субагенты — для дешёвого делегирования побочных задач, teams — для совместной работы и взаимной проверки, workflows — для repo-масштабных задач (миграции) с сотнями агентов при фикс. конкуренции (16 одновременно, до 1000 на запуск).

**Расшифровка.** Разбор по оси «кто держит план». Субагенты и скиллы оставляют управление основному агенту: он по ходу решает, что запустить, и получает обратно сжатые сводки — дёшево для второстепенных подзадач. Agent teams повышают воркеров до равноправных участников с общим списком задач и прямой перепиской — подходит для задач вроде мультиракурсного ревью. Dynamic workflows переносят оркестрацию в исполняемый JavaScript: предсказуемый, переиспользуемый скрипт, масштабируемый до сотен агентов с фиксированной конкуренцией (16 одновременно, до 1000 на запуск) — лучший выбор для repo-широких задач (миграции, аудиты). Полезно как карта выбора инструмента под конкретную задачу.

## 3. codebase-memory-mcp: чтобы агент не грепал весь репозиторий

**Анонс.** Статья на Dev.to (автор arshtechpro) представляет MCP-сервер codebase-memory-mcp — он даёт Claude Code память о структуре кодовой базы, чтобы агент не перечитывал и не грепал весь репозиторий на каждом шаге. Идея — снизить расход контекста и токенов на навигацию по большому проекту.

**Расшифровка.** MCP-сервер индексирует кодовую базу и отдаёт агенту нужные фрагменты по запросу, вместо того чтобы он каждый раз сканировал дерево файлов grep'ом. Это адресует частую проблему больших проектов: агент тратит контекст и токены на повторную навигацию. Подключается как обычный MCP-сервер к Claude Code. Полезно для работы с крупными репозиториями, где важно экономить контекст. Свежая публикация (20 июня 2026), confidence low — кейс пока без независимой проверки результатов.

## 14. Команда из «Клодов» на одной задаче: harness для аудита кода на 1,66 млн токенов

**Анонс.** Кейс из Towards Data Science (Chien Vu Minh, 12 июня 2026): автор показывает, как Claude оркеструет несколько агентов через dynamic workflows — JS-обвязку, разбивающую задачу на независимые части. На аудите кода паттерн fan-out-and-synthesize (6 finder-агентов → отдельные верификаторы → синтезатор с ранжированием) подтвердил 40 валидных проблем за ~1,66 млн токенов.

**Расшифровка.** Практический разбор dynamic workflows на реальной задаче — аудите репозитория. Схема: шесть специализированных «finder»-агентов параллельно ищут проблемы, отдельные верификаторы перепроверяют каждую находку, финальный синтезатор ранжирует результаты по серьёзности. Итог — 40 подтверждённых проблем, ~1,66 млн токенов. Ключевая мысль: «workflow переносит план в код», поэтому агент держит в голове только текущий шаг, а не весь проект — это снимает перегрузку контекста. Хороший образец того, как строить надёжные мульти-агентные прогоны.

`case-study · workflows · subagents` · Источник: [blog/Chien Vu Minh](https://towardsdatascience.com/a-harness-for-every-task-putting-a-team-of-claudes-on-one-job/)
