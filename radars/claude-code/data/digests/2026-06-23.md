## 1. State of Claude Code 2026: у 85% репозиториев есть CLAUDE.md, но субагентов — лишь у 25%

**Анонс.** Разбор 2500 публичных репозиториев с Claude Code (19 июня): CLAUDE.md есть у 84.9% (медиана ~100–150 строк), субагентов — только у 24.6%, хуков — 13.3%, project .mcp.json — 17.0%. Чем сложнее возможность, тем резче падает её освоение.

**Расшифровка.** Buildthisnow проанализировали 2500 публичных проектов. Базовая настройка почти повсеместна: CLAUDE.md у 84.9% (медиана 6.2 КБ, у 33.8% — больше 10 КБ, максимум 341 КБ). А вот продвинутые механики недоиспользованы: кастомные субагенты — у 24.6% (но у проектов с уже структурированной папкой .claude/ — до 40%), хуки — лишь у 13.3% (наименее освоенная фича), MCP-конфиг (.mcp.json) — 17.0%. Вывод авторов: рынок делится на тех, кто использует Claude Code как «автокомплит», и тех, кто строит на нём платформу. Полезно как ориентир, какие практики ещё дают конкурентное преимущество (хуки, субагенты, MCP).

## 3. Текст в «Extended Thinking» Claude Code — не настоящие рассуждения модели (HN, 237 баллов)

**Анонс.** Заметка (22 июня, 237 баллов на HN) утверждает, что строки, которые Claude Code показывает в блоке «Extended Thinking», не являются аутентичной цепочкой рассуждений модели. Повод критичнее относиться к тому, что выводится как «мышление» агента.

**Расшифровка.** Автор разбирает, что отображаемый в Claude Code «extended thinking» — это не дословный внутренний ход мысли модели, а отрендеренная/обработанная для пользователя версия, и предостерегает от того, чтобы воспринимать её как буквальный лог рассуждений. Тема вызвала большое обсуждение (170+ комментариев) о прозрачности агентов и о том, насколько можно доверять «видимому мышлению» при отладке поведения. Практический вывод: не строить отладку и доверие к агенту исключительно на тексте thinking-блока — он информативен, но не является доказательством того, что именно происходило внутри.

## 4. Recall — локальная проектная память для Claude Code (Show HN, 125 баллов)

**Анонс.** Show HN (21 июня, 125 баллов): открытый инструмент Recall даёт Claude Code персистентную локальную память по проекту — факты и контекст сохраняются между сессиями, не уходя в облако. Решает типичную боль «агент забывает наработанное».

**Расшифровка.** Recall (репозиторий raiyanyahya/recall) хранит проектную память локально, чтобы Claude Code не начинал каждую сессию «с нуля»: накопленные факты, решения и контекст переиспользуются в следующих запусках. В отличие от облачных memory-сервисов, данные остаются на машине разработчика. Как попробовать: установить из репозитория и подключить к рабочему проекту, затем проверить, переживает ли контекст перезапуск сессии. Вписывается в активную в июне тему памяти/контекста для агентов (рядом с обсуждениями CLAUDE.md и memory-записей).

`tips` · Источник: [hn/raiyanyahya](https://github.com/raiyanyahya/recall)

## 5. headroom — сжатие выводов инструментов и логов перед отправкой в LLM: −60–95% токенов

**Анонс.** Открытый headroom (22 июня) сжимает tool outputs, логи, файлы и RAG-чанки до того, как они попадут в модель: заявлено 60–95% меньше токенов «при тех же ответах». Поставляется как библиотека, прокси и MCP-сервер.

**Расшифровка.** headroom (headroomlabs-ai/headroom) атакует главный источник расхода токенов в агентных сессиях — раздутые выводы инструментов и логов. Инструмент перехватывает и компрессирует их перед подачей в LLM, обещая 60–95% экономии без потери смысла ответа. Три формата интеграции: как библиотека (встроить в свой пайплайн), как прокси (прозрачно между агентом и API) и как MCP-сервер (подключить к Claude Code напрямую). Полезно командам, упирающимся в стоимость токенов на интенсивных прогонах; цифры экономии стоит проверять на своих данных. Тема cost-tokens, активная весь июнь.

`cost-tokens · mcp` · Источник: [github/headroomlabs-ai](https://github.com/headroomlabs-ai/headroom)

## 7. Agentjacking: публичный ключ Sentry позволяет угнать Claude Code, Cursor и Codex (The New Stack)

**Анонс.** Разбор атаки agentjacking (22 июня): через публичный DSN-ключ Sentry атакующий шлёт фейковый отчёт об ошибке с вшитыми командами; когда агент читает его по MCP и «чинит баг», он их выполняет. На тестах — высокий процент успеха по трём агентам.

**Расшифровка.** The New Stack детально разбирает вектор: DSN-ключ Sentry по дизайну лежит открыто в коде сайта, поэтому атакующий без аутентификации отправляет POST с поддельным крэшем, спрятав инструкции в разделе вроде «Resolution». Разработчик просит агента починить ошибку — тот через MCP читает отчёт и исполняет встроенные команды, формально оставаясь в рамках разрешённых действий, поэтому EDR/файрволы это пропускают. Скомпрометированный агент дотягивается до env-переменных, облачных ключей и git-кредов. Защита: не давать агенту доверять данным внешних источников (Sentry, issue-трекеры) как инструкциям и изолировать MCP-инструменты, читающие пользовательский ввод. Развитие темы безопасности агентов, поднятой ранее на этой неделе.

`mcp · automation` · Источник: [hn/The New Stack](https://thenewstack.io/agentjacking-sentry-mcp-attack/)
