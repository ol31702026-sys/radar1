## 1. Claude Sonnet 5 — новый дефолтный модель в Claude Code (1M контекста, промо $2/$10)

**Зачем.** Тем, кто ежедневно кодит в Claude Code и следит за балансом цена/качество: дефолт сменился, и это влияет и на поведение, и на счёт за токены.

**Как.** Обновитесь до Claude Code ≥ 2.1.197 — Sonnet 5 подхватится как модель по умолчанию (в `/model` виден как дефолт). Нативный контекст 1M токенов. Поведение: adaptive thinking включён по умолчанию, ручной extended thinking теперь возвращает 400, нестандартные sampling-параметры тоже дают 400. Вводная цена $2/$10 за млн вход/выход — до 31 августа 2026 (далее $3/$15).

**Подвох.** Промо-цена временная. Ручной extended thinking больше не вызвать, а скрипты с кастомными sampling-параметрами сломаются на 400. Новость Anthropic не акцентирует статус дефолта именно в Claude Code — это подтверждает changelog CLI.

`cost-tokens · tips` · Источник: [blog/Anthropic](https://www.anthropic.com/news/claude-sonnet-5)

## 2. Claude Code 2.1.196: модель по умолчанию для организации, читаемые имена сессий, кликабельные вложения

**Зачем.** Командам и тим-лидам: единая модель на всю орг-учётку убирает разнобой, а читаемые имена сессий и кликабельные вложения ускоряют навигацию в длинных сессиях.

**Как.** Обновитесь до 2.1.196 (30 июня). Модель по умолчанию для организации выставляет админ в консоли — она показывается как «Org default»/«Role default» в `/model`, пока вы не выбрали свою. Имена сессий генерируются автоматически на старте. Отдельно улучшены надёжность фоновых агентов и обработка stream-watchdog.

**Подвох.** Орг-дефолт настраивает только админ в консоли — на своей стороне глобально его не переопределить, лишь вручную в `/model`. Это точечный QoL-релиз, а не крупная фича.

`tips` · Источник: [docs/Anthropic](https://code.claude.com/docs/en/changelog#2-1-196)

## 3. Claudoro — Pomodoro-таймер прямо в статуслайне Claude Code

**Зачем.** Тем, кто теряет счёт времени в длинных сессиях с агентом: таймер и прогресс-бар помодоро видно прямо в статуслайне, не отвлекаясь на сторонние приложения.

**Как.** `npm install -g claudoro && pomo setup` (нужен Node ≥ 22), затем `pomo start [минуты]`. Есть режимы minimal/classic/full, пауза/резюм/стоп/скип и лог. Отрисовка идёт «zero model round-trips» — без обращений к API, то есть без расхода токенов.

**Подвох.** Живой тик в статуслайне требует поддержки `refreshInterval` — на старых версиях Claude Code таймер не будет идти в реальном времени. Звук зависит от аудио платформы. Проект ранний (v0.1.5).

`tips` · Источник: [github/emson](https://github.com/emson/claudoro)

## 4. Опасный баг: AskUserQuestion сам продолжает работу через 60 секунд без ответа

**Зачем.** Всем, кто полагается на подтверждения (rules/hooks) перед важными действиями агента: молчаливый таймаут может пропустить шаг апрува и дать Claude действовать без вашего «да».

**Как.** Issue #73125 (2 июля): при отсутствии ответа за 60 с инструмент отдаёт «proceed using your best judgment». Параметра таймаута в схеме нет, в changelog не упомянуто. Мейнтейнер (ThariqS) подтвердил и пообещал сделать поведение opt-in. Пока — держите критичные шаги за hooks с блокирующим exit-кодом, а не только за AskUserQuestion.

**Подвох.** На момент репорта фикса ещё нет — поведение включено по умолчанию в 2.1.198. Для автоматизированных/долгих прогонов таймаут может «пробить» ваши гейты подтверждения.

`tips` · Источник: [github/tubignaaso](https://github.com/anthropics/claude-code/issues/73125)

## 5. claude-chat.nvim — Claude Code внутри Neovim без реимплементации

**Зачем.** Пользователям Neovim, которым нужен агент Claude Code без ухода в VS Code/JetBrains и без потери контекста редактора.

**Как.** Плагин `codegik/claude-chat.nvim` (99.8% Lua) запускает реальный `claude` как дочерний процесс в терминал-буфере; поднимает WebSocket-MCP (синхронизация файлов/курсора/выделения/диагностики) и HTTP-MCP (`open_file`, мосты к LSP). Установка через lazy.nvim, тоггл `<leader>ai`, отправка файла `<leader>af`; правки — диффом до применения. Нужен Neovim 0.10+ и `claude` в PATH.

**Подвох.** Это обёртка над CLI, а не нативный GUI — всё упирается в возможности терминального Claude. Пост описывает happy-path; о крайних случаях автор не пишет.

`ide-integration · mcp` · Источник: [blog/Inácio Klassmann](https://inacioklassmann.com/posts/claude-chat-nvim/)

## 6. Muxer — плагин-мультиплексор: дорогой оркестратор + дешёвые исполнители в Claude Code

**Зачем.** Тем, у кого премиум-модель жжёт бюджет на мелочах: разделение «планирование vs исполнение» по тирам режет счёт, не жертвуя качеством на сложных шагах.

**Как.** `git clone https://github.com/DangerousYams/muxer`, затем `claude plugin marketplace add ./muxer` и `claude plugin install muxer@muxer-local`. Агенты по ролям: Scout (Haiku), Writer (Sonnet), Builder/Reviewer (Opus), Oracle (Fable). Для внешних моделей — опционально `@openai/codex`, `@google/gemini-cli`. Есть отчёт по экономии.

**Подвох.** Роутинг «совещательный», жёсткие гарантии — только через пины моделей на агентов. Контекст премиум-модели всё равно тарифицируется по её ставке; расходы на внешние инструменты в отчёте не видны; цифры экономии — по прайс-листу API. Репозиторий очень молодой.

`cost-tokens · workflows` · Источник: [github/DangerousYams](https://github.com/DangerousYams/muxer)

## 7. Capacitor — общая память сессий для Claude Code, Codex и других агентов

**Зачем.** Командам, где знания теряются между сессиями и агентами: Capacitor превращает историю прогонов в общий ищущийся контекст и помогает на ревью видеть «почему», а не только дифф.

**Как.** Шесть сценариев: расшаривание ссылки на сессию в Slack и хендофф коллеге; переключение Codex↔Claude Code без потери контекста; обогащение PR-ревью рассуждениями; авто-гайдлайны по рубрикам команды; запуск браузерных агентов из дашборда; поиск по прошлым сессиям. Работает с Claude Code, Codex, Gemini CLI.

**Подвох.** Хостовые (браузерные) агенты пока только для Claude Code и Codex. Это внешний SaaS — историю сессий вы отдаёте в облако провайдера; ограничения тарифов детально не расписаны.

`mcp · workflows` · Источник: [other/Kurrent](https://capacitor.kurrent.io/)

## 8. FleetView — браузерный кокпит для десятков агентов Claude Code сразу

**Зачем.** Тем, кто гоняет «флот» агентов параллельно и теряет, какой из них ждёт подтверждения: FleetView собирает все терминалы в одном окне с умными алертами.

**Как.** Спавнит реальные шеллы через `node-pty`, рендерит их сеткой на xterm.js, держит сессии в tmux (переживают рестарт/сон). Запуск: `npm install`, `npm run doctor`, `npm run go` → `http://localhost:4280`, «+ Terminal» → выбрать папку, запустится `claude`.

**Подвох.** Только macOS/Linux. По умолчанию слушает 127.0.0.1 и БЕЗ аутентификации — любой, кто достучится до порта, выполнит команды; наружу выставлять рискованно. После ребута живые сессии Claude не восстанавливаются.

`subagents · workflows` · Источник: [github/schoppllc](https://github.com/schoppllc/Terminalcontrol)

## 9. Shipwright Harness — открытый автономный «доставщик» на базе Claude Code (MIT)

**Зачем.** Командам, которым нужен конвейер «спека → PR → деплой» на своей кодовой базе с политиками ревью, а не ручное дирижирование агентом.

**Как.** Установка плагина: `/plugin install shipwright@app-vitals/shipwright`. Быстрый старт: `git clone https://github.com/app-vitals/shipwright.git && cd shipwright && ./scripts/quickstart.sh`; дашборд метрик на `http://localhost:3460/dashboard`. Команды: `/shipwright:prd`, `:plan-session`, `:dev-task`, `:review`, `:patch`, `:deploy`. Лицензия MIT.

**Подвох.** Ранний релиз (v0.1.0). Полный стек требует Bun и go-task, а для развёртывания — Docker, tmux, PostgreSQL. Автономный деплой на своём коде — риск: нужна аккуратная настройка политик ревью.

`automation · ci-cd · workflows` · Источник: [github/app-vitals](https://github.com/app-vitals/shipwright)

## 10. От «вайб-кодинга» к agentic engineering: трёхфазный воркфлоу с Claude Code

**Зачем.** Разработчикам, которые хотят предсказуемого результата от агента, а не «как получится»: три фазы возвращают инженерную дисциплину в работу с Claude Code.

**Как.** Фаза 1 — структурированный промпт: одна императивная задача, интент, ссылки на файлы/паттерны, ограничения, что вне scope, ожидания по тестам и критерии приёмки. Фаза 2 — `/plan`: агент исследует код и составляет план, снимает неоднозначности до кода. Фаза 3 — `/code-review` + проверка покрытия тестами + walkthrough по изменениям.

**Подвох.** Это дисциплина, а не автоматика: качество зависит от того, насколько тщательно вы пишете критерии приёмки и ревьюите план. Команды `/plan` и `/code-review` должны быть доступны в вашей сборке.

`workflows · prompting` · Источник: [blog/Mehdi Raza Jaffery](https://www.apimatic.io/blog/agentic-engineering-claude-code)

## 11. VPSMaxxing — скилл, чтобы поднять Claude Code и Codex на удалённом VPS недорого

**Зачем.** Тем, кто хочет гонять агентов на выделенном VPS (постоянный аптайм, ресурсы, независимость от локальной машины), но опасается расходов и сложной настройки.

**Как.** Позиционируется как «скилл, который заставит Claude настроить ваши воркфлоу Claude Code и Codex на удалённом VPS без разорения» — то есть сам агент по скиллу проводит настройку сервера. Установка — как обычный скилл Claude Code из репозитория.

**Подвох.** README на момент сбора вытащить не удалось — конкретные шаги/скрипты не подтверждены, оцениваю по описанию репозитория. Автономная настройка сервера агентом требует осторожности с доступами и секретами. Проект нишевый.

`automation · tips` · Источник: [github/Kuberwastaken](https://github.com/Kuberwastaken/VPSmaxxing)

## 12. Видео: свой субагент Claude Code одной командой (Run the Docs)

**Зачем.** Новичкам в субагентах Claude Code: показывает самый быстрый способ завести специализированного агента, не разбираясь сразу во всей конфигурации.

**Как.** Видео демонстрирует создание кастомного субагента «одной командой» (по формату канала — через встроенный генератор/команду Claude Code). Точные шаги — в самом ролике.

**Подвох.** Это видео-туториал; расшифровку и точные команды в тексте не привожу, чтобы не выдумывать — смотрите ролик. Формат вводный, глубоких кейсов оркестрации не ждите.

## 15. Кейс: Claude Code на часах Garmin — управление сессиями с запястья

**Зачем.** Гикам и тем, кто хочет мониторить и дёргать длинные прогоны Claude Code, не сидя за ноутом — пример нетривиальной интеграции CLI-агента с носимым устройством.

**Как.** Три части: приложение на часах (Monkey C, Connect IQ), демон на Mac (Node/TS) читает транскрипты сессий из `~/.claude` и вызывает `claude` CLI, и сетевой мост через Garmin Connect + публичный HTTPS (Cloudflare Tunnel/Tailscale Funnel), т.к. у часов нет Wi-Fi. Быстрый старт без туннеля: `npm run sim`.

**Подвох.** Для реального устройства нужен публичный HTTPS-эндпоинт, сборка подписанного `.prg` и сайдлоад по MTP — не для всех. Проект экспериментальный, но код рабочий и открытый.

`case-study · automation` · Источник: [github/fashton28](https://github.com/fashton28/garmin-code)
