## 1. Claude Code 2.1.199 — цепочки навыков в одной команде и фиксы SSL/стрим-ошибок

**Зачем.** Тем, кто собрал набор своих скиллов и хочет комбинировать их одной строкой (например, `/plan /review отрефактори модуль`), не запуская по очереди. И всем, кого раздражали случайные обрывы ответа и лишние ретраи из-за SSL.

**Как.** Обновиться до 2.1.199 (`claude update` или менеджер пакетов). После этого до 5 стоящих подряд слэш-скиллов в начале сообщения загружаются вместе; SSL- и стрим-ошибки обрабатываются без потери прогресса.

**Подвох.** Это точечный патч, а не крупная фича: полезен, только если вы реально пользуетесь несколькими скиллами. Стек ограничен 5 ведущими скиллами; всё, что дальше в строке, — обычный промпт.

`skills · slash-commands · workflows` · Источник: [docs/Anthropic](https://code.claude.com/docs/en/changelog#2-1-199)

## 2. Claude Code 2.1.187 — настройка sandbox.credentials блокирует чтение секретов в песочнице

**Зачем.** Командам, гоняющим Claude Code в CI/CD или с MCP-серверами, где недоверенный ввод может спровоцировать вызов инструмента и попытку прочитать `.env`/токены. Снижает риск эксфильтрации секретов автономным агентом.

**Как.** Обновиться и включить настройку sandbox.credentials в конфиге. После этого sandboxed-команды не видят перечисленные credential-файлы и секретные env-переменные, даже если модель попробует их прочитать.

**Подвох.** Это защита только для sandboxed-команд — код, запущенный вне песочницы, по-прежнему имеет доступ к окружению. Не заменяет нормальное управление секретами (vault, ограничение прав), а лишь добавляет слой.

`ci-cd · automation · mcp` · Источник: [docs/Anthropic](https://code.claude.com/docs/en/changelog#2-1-187)

## 3. Claude Code Hooks Explained: детерминированный слой вокруг агента

**Зачем.** Тем, кто хочет жёстко (а не «попросив модель») enforced-правила: авто-форматирование после правок, блокировку опасных команд, запуск тестов на Stop, гейтинг субагентов. Хук выполняется всегда, промпт — нет.

**Как.** Хуки описываются в settings как shell-команды на событие (PreToolUse/PostToolUse/Stop/SubagentStop и др.). Управление поведением — через exit-код (0/блокировка) и JSON на stdout. В статье даны 5 конкретных реализаций для копирования.

**Подвох.** Хуки — это shell-скрипты с вашими правами: кривой хук может ломать сессию или замедлять каждый вызов инструмента. Их надо тестировать; часть из 30 событий узкоспециальны и на практике не нужны.

`hooks · automation · workflows` · Источник: [blog/Blake Crosley](https://blakecrosley.com/blog/claude-code-hooks-explained)

## 4. CLAUDE.md, Skills, Subagents, Hooks — когда что использовать

**Зачем.** Новичкам и тем, кто путается, во что оформлять знание/процесс. Правильный выбор экономит контекст и токены: не пихать в CLAUDE.md то, что должно быть скиллом по требованию, и не делать хуком то, что достаточно описать.

**Как.** Правило из статьи: всегда нужно → CLAUDE.md; вызвать по запросу → skill; должно исполниться независимо от модели → hook; нужна изоляция контекста/параллельность → subagent. Дальше — по возрастанию сложности.

**Подвох.** Это концептуальный гайд, а не пошаговая инструкция с командами: конкретику установки/синтаксиса берите из офиц. доков. Границы между примитивами на практике размываются, модель — субъективная.

`skills · subagents · hooks · tips` · Источник: [blog/speedy_devv](https://www.buildthisnow.com/blog/tools/claude-code-skills-vs-subagents-vs-hooks)

## 5. Playwright MCP + Claude Code: автогенерация и прогон браузерных тестов

**Зачем.** QA и фронтендерам, которым надоело руками писать селекторы и болванки тестов. С MCP можно сказать «зайди на страницу, кликни Blog, сделай скриншот» — и получить рабочий сценарий плюс анализ падений.

**Как.** Добавить Playwright MCP-сервер к клиенту (в Claude Code — `claude mcp add`; в статье показан вариант через `claude_desktop_config.json` для Claude Desktop) и перезапустить. Затем управлять браузером промптами и просить сгенерировать тесты.

**Подвох.** Инструкции в статье написаны под конфиг Claude Desktop, а не под CLI Claude Code — для CLI команду добавления MCP берите из доков. Не годится для строгих <100мс сценариев, жжёт токены на больших прогонах, требует доступа к облачному API.

`mcp · testing · automation` · Источник: [blog/Mykhailo Poliarush](https://testomat.io/blog/playwright-mcp-claude-code/)

**Как.** Практика из статьи: не сравнивать только ставки за Mtok — умножать на реальный рост числа токенов (тот же текст = больше токенов). Использовать промо-цену до 31.08, отслеживать фактический расход, а не прайс-лист.

**Подвох.** Оценка «+20–35%» — модельная и зависит от вашего профиля использования; на коротких задачах эффект токенизатора меньше. Материал от вендора cost-мониторинга, т.е. с очевидным интересом в теме.

**Подвох.** Это репорт/обсуждение, а не подтверждённая уязвимость с CVE и не готовое решение — часть деталей на уровне догадок. Ценность в раннем сигнале, а не в инструкции к действию.

## 8. claude-swap — переключение между несколькими аккаунтами Claude Code

**Зачем.** Тем, кто упирается в лимиты одного аккаунта и держит несколько: авто-ротация при достижении порога позволяет не простаивать. Работает и с CLI, и с VS Code-расширением.

**Как.** `uv tool install claude-swap` (или `pipx install claude-swap`). Дальше: `cswap add` — добавить текущий аккаунт, `cswap switch` — переключить, `cswap auto` — авто-переключение по лимитам, `cswap list` — расход по аккаунтам.

**Подвох.** Нужен Python 3.12+. На macOS изменения применяются с задержкой ~30с из-за кэша Keychain; для API-ключей метрики расхода не показываются; на Windows пока нет общей истории чатов между сессиями. Жонглирование аккаунтами может нарушать ToS — на свой риск.

`tips · cost-tokens` · Источник: [github/realiti4](https://github.com/realiti4/claude-swap)

## 9. Storybloq — кросс-сессионная память для Claude Code (CLI + MCP + скилл /story)

**Зачем.** Тем, кто теряет контекст между сессиями Claude Code и вручную пересказывает агенту, «на чём остановились». `.story/` коммитится в git, так что состояние проекта переживает рестарты и шарится в команде.

**Как.** `npm install -g @storybloq/storybloq@latest`, затем `storybloq setup --client all` (ставит скилл, регистрирует MCP, вешает хуки). Инициализация: `storybloq init --name "project"`; в Claude Code — `/story` (плюс `/story handover`, `/story review`, `/story auto`).

**Подвох.** Нужен Node.js 20+ и Claude Code/Codex CLI ≥0.130.0. Лицензия PolyForm Noncommercial — бесплатно для личного использования, коммерческое требует отдельной лицензии. Командный режим усложняет одновременное редактирование.

`mcp · skills · workflows` · Источник: [github/Storybloq](https://github.com/Storybloq/storybloq)

## 10. scaffolding — спец-driven плагин мульти-агентной оркестрации для Claude Code

**Зачем.** Тем, кто хочет готовый каркас spec-driven разработки, а не собирать агентов/скиллы/хуки с нуля. Хуки уже блокируют опасный `rm`, валидируют пре-коммиты и авто-форматируют правки.

**Как.** В Claude Code: `/plugin marketplace add komluk/scaffolding`, затем `/plugin install scaffolding@komluk-scaffolding` и обязательно `/reload-plugins` (иначе агенты не подхватятся). Опционально `/init-scaffolding`.

**Подвох.** Без бэкенда по умолчанию: семантическая память требует self-hosted mem0 + PostgreSQL. Файловая память ограничена 200 строками на тир. Работает только внутри Claude Code + git, standalone-использования нет.

`subagents · skills · hooks · workflows` · Источник: [github/komluk](https://github.com/komluk/scaffolding)

## 12. Видео: поставил 15 MCP-серверов, чтобы вы не ставили (9 удалил)

**Зачем.** Тем, кто хочет добавить MCP, но боится замусорить сетап: видео экономит время на самостоятельном переборе и показывает, какие серверы окупаются, а какие только раздувают контекст.

**Как.** Посмотреть, отобрать 5–6 рекомендованных MCP и добавить их через `claude mcp add`, не ставя всё подряд. Периодически ревизовать список и удалять неиспользуемые серверы.

**Подвох.** Выбор «оставить/удалить» — вкус автора под его задачи; ваш профиль может отличаться. Список MCP быстро устаревает. Это ориентир, а не истина в последней инстанции.

## 14. Кейс: автономные dev-воркфлоу на Claude Code в codecentric

**Зачем.** Тимлидам и инженерам, которым интересен не гайд, а опыт применения субагентов в реальной разработке: как разложить работу между агентами, кто кого проверяет и где это реально ускоряет.

**Как.** Подход из статьи: под задачу поднимается набор субагентов с узкими ролями (реализация, ревью, отладка), они идут параллельно и проверяют результаты друг друга перед слиянием — а не один агент делает всё.

**Подвох.** Это обзор практики конкретной команды, детали ролей и промптов обобщены; перенос на ваш проект требует адаптации. Часть выигрыша зависит от зрелости кодовой базы и тестов.

## 15. Кейс: доведение macOS-приложения перевода встреч с помощью Claude Code

**Зачем.** Тем, кто хочет увидеть Claude Code на настоящем нетривиальном приложении (аудио-захват, WebSocket, SwiftUI, системные права), а не на todo-демке — включая решение конкретных багов вроде обрыва сессии Gemini Live через ~10 минут.

**Как.** Автор применял цикл «План → реализация субагентами → AI-ревью»: Claude сперва задаёт уточняющие вопросы, пишет спецификацию, затем разводит независимых субагентов-исполнителя и ревьюера и ловит баги замкнутой проверкой. Пример: экспоненциальный backoff 2с→4с→6с + ping-keepalive для авто-reconnect.

**Подвох.** Это опыт одного разработчика на узком стеке (macOS/SwiftUI/Gemini Live API); цифр экономии времени нет. Часть решений специфична для этого приложения и напрямую не переносится.

`case-study · workflows · ide-integration` · Источник: [blog/evanlin](https://dev.to/gde/ai-in-action-refining-a-macos-meeting-translation-app-with-claude-code-auto-reconnect-floating-2856)
