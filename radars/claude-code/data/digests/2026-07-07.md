## 1. Claude Code 2.1.197–2.1.201: Sonnet 5 по умолчанию и режим прав «Manual»

**Зачем.** Тем, кто держит Claude Code обновлённым и хочет понимать, что изменилось в поведении за неделю: смена модели по умолчанию влияет на цену и лимиты, а переключение режима прав на «Manual» меняет, что выполняется без подтверждения.

**Подвох.** «Manual» по умолчанию = больше подтверждений в интерактиве; для скриптов/CI задавайте permission mode явно. Промо-цена Sonnet 5 временная (до 31.08).

## 2. Сабагенты в фоне по умолчанию и авто-хендофф в draft PR (v2.1.198)

**Зачем.** Тем, кто гоняет длинные задачи и параллельных агентов: фоновые сабагенты не блокируют основную сессию, а авто-открытие draft PR превращает «агент доделал» в готовый к ревью PR без ручных шагов.

**Как.** Обновиться (`claude update`, v2.1.198+). Сабагенты по умолчанию уходят в фон, приходит уведомление по завершении; фоновые агенты из `claude agents` в конце сами делают commit/push и открывают draft PR. Встроенный `/dataviz` подгружает правила выбора типа графика, вёрстки и проверку контраста палитры — установка не нужна.

**Подвох.** Агенты, которые сами пушат и открывают PR, требуют аккуратных прав на репозиторий; для «Claude in Chrome» нужен браузер-коннектор.

## 3. Anthropic: self-hosted шлюз для Claude Code на AWS Bedrock и Google Cloud

**Зачем.** Платформенным/DevOps-командам, разворачивающим Claude Code на десятки-сотни разработчиков без «россыпи» долгоживущих ключей на машинах и с централизованным контролем трат.

**Как.** Развернуть stateless-контейнер шлюза (PostgreSQL) в своей инфраструктуре; он работает как OIDC relying party (Google Workspace, Entra ID, Okta), выдаёт короткоживущие сессии, раздаёт политики при входе, шлёт телеметрию по OTLP и маршрутизирует трафик на Claude API, Bedrock или Google Cloud с фейловером. Лимиты трат — дневные/недельные/месячные на уровне org/группы/пользователя.

**Подвох.** Это инфраструктурный компонент: нужен свой PostgreSQL и эксплуатация контейнера; ценность в основном для команд/энтерпрайза, не для соло-разработчика.

`automation · ci-cd · tips` · Источник: [blog/DevOps.com](https://devops.com/anthropic-adds-enterprise-gateway-to-simplify-claude-code-access-on-aws-and-google-cloud/)

## 4. contextrot: измеряет «гниение контекста» Claude Code по вашим же логам сессий

**Зачем.** Тем, кто замечает, что Claude Code «тупеет» к концу длинной сессии, но не знает, где именно порог — инструмент меряет это на ваших данных, а не на лабораторных бенчмарках.

**Как.** `uvx contextrot` (или `pip3 install contextrot`, Python 3.9+). Читает локальные JSONL-логи и коррелирует пять сигналов сбоя с заполнением окна; выдаёт вердикт, есть ли у вашего сетапа context rot и во что он обходится.

**Подвох.** Наблюдательный, не причинный анализ (корреляция ≠ причина); работает только с агентами, ведущими локальные JSONL (Claude Code, OpenCode); v0.3.0 от 06.07.2026, ранняя стадия.

`cost-tokens · tips` · Источник: [github/PriyanshuDoshi](https://github.com/Priyanshu-byte-coder/contextrot)

## 5. claude-cache-guard: не жечь токены на перечитывание сессии после сброса окна

**Зачем.** Тем, кто упирается в лимиты и теряет кредиты на том, что при возврате в большую сессию весь диалог перечитывается как некэшированный ввод ещё до начала работы.

**Как.** `npm install -g claude-cache-guard`, затем `ccg install`; включить в проекте `/ccgenable`, после сброса окна продолжить `/ccgresume` в НОВОЙ сессии. Метрики хранятся только локально.

**Подвох.** После установки нужно перезапустить Claude Code (подгрузить слэш-команды); `/ccgresume` работает только в свежей сессии — вернётесь в старую и смысл теряется. Проект молодой (8★, 3 коммита).

`cost-tokens` · Источник: [github/yuliang615](https://github.com/yuliang615/claude-cache-guard)

## 6. memhub: постоянная память проекта для Claude Code с ручным одобрением записей

**Зачем.** Тем, кто устал каждую сессию заново объяснять агенту команды сборки и архитектурные решения, но не хочет доверять агенту бесконтрольную запись в память.

**Как.** Для Claude Code: попросить «Please install memhub for me, then turn on hybrid recall» и следовать шагам; либо вручную `git clone`, `cargo install --path ~/src/memhub --force`, `memhub init` в проекте. Есть гибридный recall (семантика + ключевые слова), локатор кода `memhub locate "retry logic"`, опц. синк через Google Drive.

**Подвох.** Одно-пользовательский (last-writer-wins при синке); эмбеддинги дают ~50 мс на запись (есть FTS-only режим); при сборке качается модель ~218 МБ.

`mcp · tips` · Источник: [github/kninetimmy](https://github.com/kninetimmy/memhub)

## 7. ccc: кастомный лаунчер Claude Code со слоями конфигов и хуками на TypeScript

**Зачем.** Тем, кто держит несколько проектов/пресетов и хочет переопределять промпты, команды, агентов и MCP послойно, не копируя конфиги руками.

**Как.** `git clone https://github.com/3rd/ccc.git`, `bun install`, `bun link`, `bun add -g tsx`; дальше запускать `ccc` вместо `claude`. Конфиги мёржатся global→preset→project; хуки и MCP пишутся на TS через хелперы.

**Подвох.** Нужны Bun и tsx в PATH; не тестировался на Windows (автор просит заводить issue). 17★, MIT.

`hooks · mcp` · Источник: [github/3rd](https://github.com/3rd/ccc)

## 13. peek-cli: дать Claude Code «глаза» в браузере через скриншоты вкладок

**Как.** `/plugin marketplace add puffinsoft/peek-cli`, затем `/plugin install view-browser-tab@peek-cli`; запустить демон `peeked start` и подключить браузер через Chrome-расширение. Агент получает только скриншоты по запросу.

**Подвох.** Только просмотр — ни кликов, ни ввода (это и фича безопасности, и ограничение); нужен Chrome-экстеншн и запущенный демон. v0.2.0 от 03.07.2026, 45★, ранний.
