## 1. pi-dynamic-workflows: динамические воркфлоу в стиле Claude Code для агента Pi

**Зачем.** Тем, кто хочет оркестрировать много субагентов детерминированно (fan-out, adversarial-review, deep-research), но работает в агенте Pi, а не в Claude Code.

**Как.** `pi install npm:@quintinshaw/pi-dynamic-workflows`, затем `/reload` — появляются инструмент `workflow` и команды `/workflows`, `/deep-research`, `/adversarial-review`. В скрипте `agent()`, `parallel()`, `pipeline()`, `phase()`; до 16 агентов параллельно / 1000 всего. Прерванный прогон доигрывается из журнала (готовые агенты не перезапускаются — токены не тратятся). `isolation: 'worktree'` даёт агенту свою ветку. Есть `/ultracode`.

**Подвох.** Это для Pi, а не для Claude Code напрямую; польза зависит от того, используете ли вы Pi. Молодой проект (v2.12.0 от 09.07.2026).

`workflows · subagents` · Источник: [github/QuintinShaw](https://github.com/QuintinShaw/pi-dynamic-workflows)

## 5. Как урезать «раздутый» системный промпт Claude Code и сэкономить токены

**Зачем.** Тем, кто платит за токены на каждом ходу и хочет уменьшить размер запроса, отключив неиспользуемые фичи и инструменты.

**Как.** `/context` показывает разбивку контекста; логирующий прокси (`proxy.mjs`) ранжирует инструменты по размеру. В `~/.claude/settings.json` включить `disableBundledSkills`, `disableWorkflows`, а лишние инструменты добавить в `permissions.deny` «голым именем» (напр. `NotebookEdit`). Нюанс: голое имя удаляет определение инструмента из payload, а scoped-правило лишь блокирует вызов, оставляя описание.

**Подвох.** Отключив скиллы/воркфлоу, теряете эти возможности — балансируйте экономию и функциональность. Дата на странице явно не указана; взято по появлению на Hacker News (09.07.2026).

`cost-tokens · prompting` · Источник: [blog/aihero.dev](https://www.aihero.dev/how-to-kill-the-bloat-in-claude-codes-system-prompt)

## 6. claude-agent-team: локальный governance-слой Claude Code на его же примитивах

**Зачем.** Тем, кому нужен аудит, память и контроль расходов над Claude Code без внешних сервисов — всё локально.

**Как.** Собрано на нативных примитивах (hooks, subagents, skills, MCP). Каждая сессия — запрашиваемый локальный SQLite-журнал; по нему система предсказывает следующий диспатч, поднимает прошлые инциденты, считает стоимость по сессиям и гейтит коммиты.

**Подвох.** Молодой проект (★5), концепт впереди зрелости; «предсказание следующего вызова» — экспериментальная фича. Обкатывайте на некритичных репозиториях.

`subagents · hooks · mcp · skills` · Источник: [github/ek33450505](https://github.com/ek33450505/claude-agent-team)

## 7. Приём /fork в Claude Code: разветвить сессию, не теряя контекст

**Зачем.** Тем, кто хочет проверить альтернативный подход, не разрушая основную сессию и её контекст.

**Как.** Судя по ролику, на нужном шаге вызывается `/fork`, и дальше работа идёт в ветке-копии сессии. Формат — короткое видео, конкретика ограничена демонстрацией.

**Подвох.** Короткий ролик без глубокого разбора; убедитесь, что `/fork` доступен в вашей версии Claude Code.
