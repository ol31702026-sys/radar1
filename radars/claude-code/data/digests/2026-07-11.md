## 1. Claude Code 2.1.203–2.1.206: /doctor-чекап, защита транскриптов, автообновление фоновых агентов

**Зачем.** Тем, кто живёт в Claude Code ежедневно: за неделю закрыли набор болячек — залипание `--resume`/`--continue` на старте, игнорирование `request_timeout_ms` у MCP-серверов, нестабильные фоновые сессии на Windows и macOS.

**Как.** Обновиться (`claude update`) и прогнать `/doctor` (алиас `/checkup`) — теперь это полная диагностика установки; в 2.1.206 он же предлагает вычистить из закоммиченного CLAUDE.md то, что Claude и так выводит из кода. `/cd` подсказывает пути как `/add-dir`; `/commit-push-pr` сам разрешает `git push` в настроенный push remote; auto mode блокирует подделку транскриптов сессий.

**Подвох.** Больших новых фич нет — серия стабилизационная; часть изменений (login через Anthropic-шлюзы) касается только enterprise-развёртываний.

`tips · automation` · Источник: [github/Anthropic](https://github.com/anthropics/claude-code/releases/tag/v2.1.206)

## 2. VS Code 1.128: несколько чатов в одной Claude-сессии — форки, параллельные ходы, subagent-треки

**Зачем.** Тем, кто работает с Claude в VS Code и хочет сравнить два подхода к задаче или вести параллельные ветки работы, не плодя несвязанные сессии.

**Как.** Обновить VS Code до 1.128 (вышел 8 июля). В окне Agents: добавить чат в сессию, форкнуть чат от раннего хода, слать ходы конкурентно; у каждого чата своя история, заголовок и модель. Работа субагентов видна как read-only peer-чаты.

**Подвох.** Работает через agent host — нужен включённый `chat.agentHost.enabled`; в корпоративных установках флаг контролирует администратор.

`ide-integration · subagents` · Источник: [docs/VS Code team](https://code.visualstudio.com/updates/v1_128)

## 3. Anthropic показала встроенный браузер Claude Code в десктоп-приложении

**Зачем.** Проверять свёрстанное и прокликивать своё веб-приложение прямо из сессии, не поднимая Playwright или browser-MCP: Claude открывает страницу, читает её и взаимодействует с элементами.

**Как.** В Claude Code Desktop попросить агента открыть URL — доки, дизайн, локальный dev-сервер. Работает в песочнице отдельно от личного браузера; персистентность сессий настраивается.

**Подвох.** Профиль без ваших логинов — для сайтов с авторизацией Anthropic рекомендует расширение Claude in Chrome, которое делит состояние логина с основным браузером.

`testing · tips` · Источник: [blog/9to5Mac](https://9to5mac.com/2026/07/10/anthropic-highlights-claude-codes-in-app-browser-on-the-desktop/)

## 4. Abralo: несколько агентов Claude Code в одном окне — бесплатный десктоп-менеджер

**Зачем.** Тем, кто гоняет 3–5 параллельных сессий Claude Code и устал от зоопарка терминальных вкладок: одно окно показывает статусы всех агентов и каким нужно внимание.

**Как.** Скачать с abralo.com (несколько МБ), войти со своей подпиской Claude Pro/Max — отдельная регистрация не нужна. Схема «один агент — одна папка проекта»; токен-аналитика с burndown и предупреждениями о лимитах.

**Подвох.** Нужна активная подписка Claude; изоляции через git worktree нет — агенты работают прямо в папках проектов. Пуш-уведомления на телефон — уже в платной Pro-версии.

`workflows · tips` · Источник: [hn/cwbuilds](https://abralo.com/)

## 5. Atelier: open-source-рантайм режет расход Claude Code — в среднем −30% стоимости

**Зачем.** Тем, кто упирается в лимиты подписки или жжёт API-бюджет на больших репозиториях: экономия без смены привычек — оптимизация невидима.

**Как.** `curl -fsSL https://install.atelier.ws | bash`, затем `atelier init` в проекте (~30 секунд). Бенчмарки с закоммиченными прогонами: SWE-bench Verified −29.5% стоимости, exploration-задачи до −67%. Есть локальный анализ экономии до установки.

**Подвох.** Молодой проект (v0.3.9 от 10 июля); авторы честно показывают случаи без выигрыша (Terminal-Bench: −0.2 п.п. точности). Установка через `curl | bash` — прочитайте скрипт.

`cost-tokens` · Источник: [github/atelier-ws](https://github.com/atelier-ws/atelier)

## 6. agent-convert: продолжить сессию Claude Code в Codex (и наоборот) одной командой

**Зачем.** Если задача начата в Claude Code, а докручивать удобнее в Codex (или упёрлись в недельный лимит) — перенос сессии вместо пересказа контекста с нуля.

**Как.** `npm i -g agent-convert`, затем `agent-convert ~/.claude/projects/<project>/<session-id>.jsonl --to codex` — сессия конвертируется и сразу устанавливается. Формат источника определяется автоматически; всё локально, без сети.

**Подвох.** Конверсия «structure-preserving, но не всегда lossless»; Windows не тестирован; `--dry-run` пока нет.

`tips · workflows` · Источник: [github/theoriclabs](https://github.com/theoriclabs/agent-convert)

## 8. Видео: MCP или Skills — что выбирать под задачу в Claude Code

**Зачем.** Тем, кто навешивает MCP-серверы на всё подряд и удивляется расходу контекста — или, наоборот, пытается скиллом решить задачу интеграции с внешней системой.

**Как.** Базовый ориентир: повторяемая процедура или знание → Skill (SKILL.md подгружается по требованию); живой доступ к данным и действиям во внешних системах (БД, API, браузер) → MCP. Аргументация и примеры — в ролике.

**Подвох.** Границы размываются: скиллы могут дёргать CLI-инструменты, а MCP-серверы — отдавать знания; видео-рассуждение быстро не отсканировать.

## 10. «Claude изменился»: как пользоваться экосистемой Claude по-новому в июле 2026

**Зачем.** Тем, кто использует Claude как чат «спросил — получил» и чувствует, что недоиспользует подписку: карта, какие части экосистемы для чего.

## 11. 5 скиллов Claude Code, которые стоит поставить: Superpowers, Impeccable, Security Review и другие

**Зачем.** Быстро прокачать свежепоставленный Claude Code чужой экспертизой вместо того, чтобы месяцами набивать собственные шишки.

**Как.** Superpowers: `/plugin install superpowers@claude-plugins-official`. Impeccable: `npx impeccable install` + `/impeccable init`. ## 13. Кейс: промпт-бар на своём сайте — Claude Code правит и деплоит сайт прямо с этой же страницы

**Зачем.** Владельцам собственных сайтов и лендингов: мелкие правки (тексты, стили, секции) текстом, голосом или даже фото прямо со страницы — без IDE и без подрядчика.

**Как.** Self-hosted-набор: виджет в Shadow DOM + Express-сервер на Fly.io + headless Claude Code в выделенной VM + git push → автодеплой (Netlify/Vercel/CF Pages). Нужны подписка Claude Pro/Max, GitHub PAT, бесплатный Supabase и Fly.io. Релиз v0.1.0, показан на Show HN.

**Подвох.** Сетап нетривиален — пять сервисов; в публичном репо промпты становятся публичными commit-сообщениями; правки уезжают в прод без ревью.

`case-study · automation` · Источник: [github/vibhasjain](https://github.com/vibhasjain/agent-keyboard)
