## 1. agent-security-scanner-mcp: MCP-фаервол против prompt-инъекций и выдуманных пакетов

**Зачем.** Тем, кто разрешает Claude Code ставить пакеты и писать код в реальный проект и боится, что агент подхватит выдуманную зависимость или отработает вредоносную инструкцию из README/issue.

**Как.** Установка одной командой: `npx agent-security-scanner-mcp init claude-code`, затем перезапустить Claude Code. Даёт: детект prompt-инъекций (59+ правил + защита от обфускации и мультикодировки), проверку имён пакетов против базы 4.3M+ (npm/PyPI/RubyGems и ещё 5 экосистем), 1700+ правил уязвимостей (SQLi/XSS/command injection) через AST и taint-анализ, 120 шаблонов авто-фиксов, аудит самих MCP-серверов (tool-name spoofing, rug-pull) и генерацию SBOM (CycloneDX) с проверкой по OSV.dev. Версия 4.4.12 от 12 июля.

**Подвох.** Только статический анализ — код не исполняет, рантайм-атаки не ловит; нужен Node 18+ и Python 3.x; полноценный детект галлюцинаций npm тянет пакет побольше (10.3 МБ против 2.7 МБ базового).

## 2. Как срезали счёт за токены Claude Code на 77%: прокси-профайлер и обрезка старых чтений

**Зачем.** Командам и соло-разработчикам, которые жгут API-бюджет на длинных сессиях с большими репозиториями и хотят понять, за что именно платят.

**Как.** Трафик пропускается через прокси `aap` (AI Agent Profiler): `aap serve --optimize`, затем `aap run claude`. Ключевые приёмы: `pruneStale` — старые прочитанные файлы сворачиваются в заглушку вида `[read scheduler.js earlier — 47 lines]` (~57% экономии); `pruneUnusedTools` — выкидываются схемы неиспользуемых инструментов (~23%); переустановка 4 cache-breakpoint'ов Anthropic после правок, чтобы не терять попадания в кэш; дедуп чтений и обрезка больших выводов.

**Подвох.** Приём завязан на явный prompt-кэш Anthropic — на implicit-кэше (например, DeepSeek) те же трюки не сработали. Нужно вклинить прокси в инфраструктуру агента — не в каждом окружении тривиально.

`cost-tokens` · Источник: [blog/rguiu](https://dev.to/rguiu/what-i-learned-cutting-claude-codes-token-bill-by-77-3ef)

## 4. Confessor: разбор логов Claude Code — что агент читал и не утекло ли в сеть

**Зачем.** Разработчикам, которые дают агенту доступ к машине и хотят пост-фактум проверить, не читал ли он `.env`/SSH-ключи и не ушло ли что-то в сеть.

**Как.** Запуск: `npx confessor` (флаги `--json`, `--out <file>`, `--fail-on critical|high|medium`; можно скормить экспорт чата: `npx confessor ~/Downloads/export.zip`). Читает JSON-логи сессий из `~/.claude/projects`, классифицирует файлы по чувствительности, ищет секреты (30 паттернов токенов, 13 — PII), сетевые «стоки» (curl/wget/MCP-вызовы) и коррелирует «прочитал секрет → сделал сетевой вызов». Значения ключей маскируются (`sk_live_…MPLE`); без зависимостей и без обращений в сеть.

**Подвох.** Работает ретроспективно, без перехвата в реальном времени; правила эвристические (новый формат секрета может проскочить); «путь утечки» — повод разобраться, а не доказательство кражи. Проект совсем новый (пока 1 звезда).

## 7. Свой первый MCP-сервер для Claude Code: read-only доступ к БД, где правило держит инструмент, а не промпт

**Зачем.** Тем, кто вручную носит данные между Claude и своей БД/инфраструктурой и хочет дать агенту структурированный безопасный доступ, а не сырой shell.

**Как.** На FastMCP собирается один инструмент — read-only запрос к Postgres: подключение через урезанную роль, транзакции только на чтение, таймаут запроса и лимит строк (500). Идея автора: «инструмент обеспечивает правило, а не промпт» — read-only tool не сможет уронить таблицу, как бы ни повернулся диалог. Совет: начинать с наблюдательных read-only инструментов, потом добавлять запись; кастомные MCP можно комбинировать с готовыми.

**Подвох.** Это вводный пример (часть 4 серии) — до продакшена нужно докрутить аутентификацию, аудит и обработку ошибок; безопасность держится на правах в БД, а не на самом MCP-сервере.

`mcp` · Источник: [blog/gde03](https://dev.to/gde03/claude-code-beyond-the-prompt-part-4-your-first-mcp-server-give-claude-safe-hands-on-your-own-b8p)

## 9. Code Airlock: запуск Claude Code в одноразовых microVM с ревью коммитов перед merge

**Зачем.** Тем, кто хочет дать агенту работать автономно (ставить пакеты, править файлы, гонять тесты) без риска для рабочего дерева, с ревью-гейтом перед вливанием.

**Как.** Установка: `npm install -g code-airlock` (или `brew`/shell-installer). Нужны Docker Sandboxes (`sbx` CLI) и KVM. Дальше `code-airlock doctor` (проверка), `code-airlock up` (поднять песочницу с Claude Code), `code-airlock run "add pagination and run tests"`, затем `code-airlock diff` и `code-airlock merge`. Агент коммитит в приватный клон внутри microVM, хост-репо защищён; вы фетчите коммиты песочницы, смотрите диффы и мерджите локально. v0.2.0 от 11 июля.

**Подвох.** Требует Docker Sandboxes (Linux+KVM, macOS Apple Silicon, Windows 11 WSL2) — Intel-макам нельзя; незакоммиченные правки в песочнице обратно не приезжают; сетевая политика применяется глобально ко всем песочницам.

`automation · workflows` · Источник: [github/Trivo25](https://github.com/Trivo25/code-airlock)

## 11. Claude Code /doctor: чинит настройку и считает контекст-косты скиллов, MCP и плагинов

**Зачем.** Тем, у кого раздулся `~/.claude` (десятки скиллов/MCP/плагинов) и контекст съедается уже на старте сессии: даёт способ почистить настройку без ручного аудита.

**Как.** `/doctor` (алиас `/checkup`) в любой сессии: проверяет здоровье установки, находит неиспользуемые скиллы, MCP-серверы и плагины против их стоимости в контексте, дедуплицирует локальные `CLAUDE.md` против закоммиченных, предлагает срезать из `CLAUDE.md` то, что модель и так выведет из кода, флагит медленные хуки. Сначала показывает находки, меняет только после подтверждения (появилось в v2.1.205, неделя 6-10 июля). Заодно: десктоп-приложение получило встроенный песочный браузер (Claude сам открывает доки/дизайны и кликает по страницам), а auto mode теперь блокирует подмену файлов транскрипта и спрашивает перед `rm -rf` по неразрешённой переменной.

**Подвох.** /doctor правит конфиг и `CLAUDE.md` — перед подтверждением стоит глянуть, что именно срезается (эвристика может предложить убрать нужное); встроенный браузер — только в десктоп-приложении, не в CLI; ветка стабилизационная, крупных пользовательских новинок мало.

`tips · cost-tokens · ide-integration` · Источник: [docs/Anthropic](https://code.claude.com/docs/en/whats-new/2026-w28)

## 14. Кейс: инди-разработчик собрал Shopify-приложение на Claude Code через спеки — и честный разбор выручки $9.99

**Зачем.** Инди-мейкерам и соло-основателям, которые делают нишевый SaaS в свободное время и хотят понять реальную планку — и по скорости, и по выручке.

**Как.** Подход spec-driven: вместо написания кода автор пишет подробные спецификации (200+ строк — модели данных, крайние случаи, обработка ошибок, безопасность), а Claude Code реализует; автор ревьюит каждую строку и сам принимает все архитектурные решения (какие GraphQL-мутации, как устроен биллинг). Обычно 3 круга ревью до релиза — оно поймало 3 критичных бага: неверную базу подсчёта, отсутствие idempotency-гардов и мягкие лимиты вместо жёстких. Стек: React Router v7, Shopify App Remix, Polaris, Neon Postgres, Prisma, GCP Cloud Run. Тарифы: Free (безлимитный экспорт, 50 строк импорта навсегда), Pro $9.99/мес за 10k строк. Сборка — «несколько недель по вечерам», инфраструктура ~$0/мес.

**Подвох.** Отрезвляющая экономика: единственный платный клиент удалил приложение через 2 часа, выручка — $9.99. Spec-first требует дисциплины — Claude пишет код, но качество держится на ревью каждой строки и на том, что архитектуру автор держит в своих руках.

`case-study · workflows` · Источник: [blog/curythm](https://dev.to/curythm/building-a-shopify-app-with-claude-code-spec-driven-development-and-pricing-design-35pl)

## 15. Кейс: как малые диджитал-агентства режут рутину на Claude Code — отчёт за 40 минут вместо 2 дней

**Зачем.** Владельцам небольших агентств и account-менеджерам, у кого маржу съедает ручная рутина (отчёты, онбординг, скаффолдинг сайтов), а заказная автоматизация стоит пятизначных сумм.

**Как.** Пять направлений с цифрами. (1) Кастомная клиентская отчётность — скрипты интеграции с API + авто-нарратив: двухдневный месячный цикл младшего аналитика → 40 минут, окупается за первый же цикл. (2) Внутренние инструменты (напр. авто-онбординг, убирающий 15-шаговые чек-листы) — вместо заказной разработки за $50k+ и месяцев собираются за дни/недели. (3) Ускорение разработки: фронтенд-скаффолдинг 90 мин → 20 мин ревью, разработчик тянет 3→5 проектов в квартал. (4) Контент-операции: пайплайны «бриф → черновик», аудиты, адаптация под каналы. (5) Собственные ИИ-инструменты как конкурентный ров.

**Подвох.** Статья честно предупреждает: «ускорение» требует продуманного направления и дисциплины ревью; код нужно тестировать, версионировать и проверять человеком до прода; часть junior-задач сжимается — людей надо переводить на более ценную работу.
