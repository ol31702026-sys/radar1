## 1. Claude Code 2.1.207: авто-режим по умолчанию на Bedrock/Vertex/Foundry и защита плагин-хуков от инъекций

**Зачем.** Тем, кто гоняет Claude Code на облачных провайдерах (Bedrock/Vertex/Foundry), и тем, кто ставит сторонние плагины: релиз убирает лишний env-флаг для auto mode и закрывает реальную дыру в плагинах.

**Как.** Обновиться (`claude update` или `npm i -g @anthropic-ai/claude-code`). Auto mode на Bedrock/Vertex/Foundry больше не требует `CLAUDE_CODE_ENABLE_AUTO_MODE`; модель по умолчанию на AWS и Vertex — Opus 4.8. Подстановка `${user_config.*}` в shell-формате хуков/мониторов/MCP-настроек теперь отклоняется, чтобы значения из конфига не утекали в шелл. Плюс фикс залипания терминала на длинных списках/таблицах и починка «молчаливого» согласия на remote-management в неинтерактивном режиме. Всего ~24 изменения.

**Подвох.** Серия по-прежнему стабилизационная — крупных пользовательских фич нет; смена модели по умолчанию на AWS/Vertex на Opus 4.8 может неожиданно поднять расход токенов, если вы рассчитывали на Sonnet.

## 2. Как срезали счёт за токены Claude Code на 77%: прокси-профайлер и обрезка старых чтений

**Зачем.** Командам и соло-разработчикам, которые жгут API-бюджет на длинных сессиях с большими репозиториями и хотят понять, за что именно платят.

**Как.** Трафик пропускается через прокси `aap` (AI Agent Profiler): `aap serve --optimize`, затем `aap run claude`. Ключевые приёмы: `pruneStale` — старые прочитанные файлы сворачиваются в заглушку вида `[read scheduler.js earlier — 47 lines]` (~57% экономии); `pruneUnusedTools` — выкидываются схемы неиспользуемых инструментов (~23%); переустановка 4 cache-breakpoint'ов Anthropic после правок, чтобы не терять попадания в кэш; дедуп чтений и обрезка больших выводов. Итог: вход 3.9M → 776K токенов, цена $2.04 → $0.48 при 54/54 пройденных тестах.

**Подвох.** Приём завязан на явный prompt-кэш Anthropic — на implicit-кэше (например, DeepSeek) те же трюки не сработали. Нужно вклинить прокси в инфраструктуру агента — не в каждом окружении тривиально.

`cost-tokens` · Источник: [blog/rguiu](https://dev.to/rguiu/what-i-learned-cutting-claude-codes-token-bill-by-77-3ef)

## 3. agent-security-scanner-mcp: MCP-фаервол против prompt-инъекций и выдуманных пакетов

**Зачем.** Тем, кто разрешает Claude Code ставить пакеты и писать код в реальный проект и боится, что агент подхватит выдуманную зависимость или отработает вредоносную инструкцию из README/issue.

**Как.** Установка одной командой: `npx agent-security-scanner-mcp init claude-code`, затем перезапустить Claude Code. Даёт: детект prompt-инъекций (59+ правил + защита от обфускации и мультикодировки), проверку имён пакетов против базы 4.3M+ (npm/PyPI/RubyGems и ещё 5 экосистем), 1700+ правил уязвимостей (SQLi/XSS/command injection) через AST и taint-анализ, 120 шаблонов авто-фиксов, аудит самих MCP-серверов (tool-name spoofing, rug-pull) и генерацию SBOM (CycloneDX) с проверкой по OSV.dev. Версия 4.4.12 от 12 июля.

**Подвох.** Только статический анализ — код не исполняет, рантайм-атаки не ловит; нужен Node 18+ и Python 3.x; полноценный детект галлюцинаций npm тянет пакет побольше (10.3 МБ против 2.7 МБ базового).

`mcp` · Источник: [github/sinewaveai](https://github.com/sinewaveai/agent-security-scanner-mcp)

## 4. Свой первый MCP-сервер для Claude Code: read-only доступ к БД, где правило держит инструмент, а не промпт

**Зачем.** Тем, кто вручную носит данные между Claude и своей БД/инфраструктурой и хочет дать агенту структурированный безопасный доступ, а не сырой shell.

**Как.** На FastMCP собирается один инструмент — read-only запрос к Postgres: подключение через урезанную роль, транзакции только на чтение, таймаут запроса и лимит строк (500). Идея автора: «инструмент обеспечивает правило, а не промпт» — read-only tool не сможет уронить таблицу, как бы ни повернулся диалог. Совет: начинать с наблюдательных read-only инструментов, потом добавлять запись; кастомные MCP можно комбинировать с готовыми.

**Подвох.** Это вводный пример (часть 4 серии) — до продакшена нужно докрутить аутентификацию, аудит и обработку ошибок; безопасность держится на правах в БД, а не на самом MCP-сервере.

`mcp` · Источник: [blog/gde03](https://dev.to/gde03/claude-code-beyond-the-prompt-part-4-your-first-mcp-server-give-claude-safe-hands-on-your-own-b8p)

## 5. MenteDB: постоянная память для Claude Code, переживающая сжатие контекста

**Зачем.** Тем, кто теряет важные детали проекта после авто-сжатия или `/clear` и устал держать всё в раздутом CLAUDE.md.

**Как.** Установка: `npx mentedb-mcp@latest setup` — по умолчанию ставит хуки Claude Code и инструкции агента; для синка между устройствами `npx mentedb-mcp@latest setup login`; удалённо — HTTP-эндпоинт `https://api.mentedb.com/mcp`. Под капотом — Rust-БД с эмбеддингами и вектор-поиском, которая сама извлекает факты и подмешивает в контекст только нужные.

**Подвох.** Бесплатный тариф — 100 воспоминаний и 200 ходов/мес (Pro $19/мес — безлимит и 50k ходов); для фонового обогащения нужен свой LLM-провайдер; это ещё один сервис в стеке.

`mcp · hooks` · Источник: [hn/mentedb](https://mentedb.com)

## 7. ClaudeLines: галерея кастомных статус-строк для Claude Code — контекст, стоимость и git в подвале

## 8. Code Airlock: запуск Claude Code в одноразовых microVM с ревью коммитов перед merge

**Зачем.** Тем, кто хочет дать агенту работать автономно (ставить пакеты, править файлы, гонять тесты) без риска для рабочего дерева, с ревью-гейтом перед вливанием.

**Как.** Установка: `npm install -g code-airlock` (или `brew`/shell-installer). Нужны Docker Sandboxes (`sbx` CLI) и KVM. Дальше `code-airlock doctor` (проверка), `code-airlock up` (поднять песочницу с Claude Code), `code-airlock run "add pagination and run tests"`, затем `code-airlock diff` и `code-airlock merge`. Агент коммитит в приватный клон внутри microVM, хост-репо защищён; вы фетчите коммиты песочницы, смотрите диффы и мерджите локально. v0.2.0 от 11 июля.

**Подвох.** Требует Docker Sandboxes (Linux+KVM, macOS Apple Silicon, Windows 11 WSL2) — Intel-макам нельзя; незакоммиченные правки в песочнице обратно не приезжают; сетевая политика применяется глобально ко всем песочницам.

`automation · workflows` · Источник: [github/Trivo25](https://github.com/Trivo25/code-airlock)

## 9. Confessor: разбор логов Claude Code — что агент читал и не утекло ли в сеть

**Зачем.** Разработчикам, которые дают агенту доступ к машине и хотят пост-фактум проверить, не читал ли он `.env`/SSH-ключи и не ушло ли что-то в сеть.

**Как.** Запуск: `npx confessor` (флаги `--json`, `--out <file>`, `--fail-on critical|high|medium`; можно скормить экспорт чата: `npx confessor ~/Downloads/export.zip`). Читает JSON-логи сессий из `~/.claude/projects`, классифицирует файлы по чувствительности, ищет секреты (30 паттернов токенов, 13 — PII), сетевые «стоки» (curl/wget/MCP-вызовы) и коррелирует «прочитал секрет → сделал сетевой вызов». Значения ключей маскируются (`sk_live_…MPLE`); без зависимостей и без обращений в сеть.

**Подвох.** Работает ретроспективно, без перехвата в реальном времени; правила эвристические (новый формат секрета может проскочить); «путь утечки» — повод разобраться, а не доказательство кражи. Проект совсем новый (пока 1 звезда).

`tips` · Источник: [github/ninjahawk](https://github.com/ninjahawk/Confessor)

## 12. Видео: почему не стоит давать Claude Code авто-сжимать контекст

**Зачем.** Тем, кто замечает, что после авто-сжатия Claude «забывает» договорённости и качество падает.

**Как.** Идея ролика (канал James Goldbach): не полагаться на авто-compact, а самому резать и переносить контекст — вовремя делать `/clear`, держать ключевое в заметках/файлах, начинать свежую сессию под новую задачу. Перекликается с приёмами постоянной памяти (см. находку про MenteDB).

**Подвох.** Это практика и мнение автора, а не измеренный бенчмарк; ручное управление контекстом требует дисциплины.

## 14. Кейс: соло-разработчик собрал на Claude Code автономного агента для отклика на вакансии

**Зачем.** Фрилансерам и соло-специалистам как образец, как из Claude Code собрать реального автономного помощника под рутину (здесь — отклики на вакансии), а не просто чат.

**Как.** `gh repo fork MadsLorentzen/ai-job-search --clone`, поставить инструменты через Bun, дальше команды `/setup` (профиль), `/scrape` (поиск по job-порталам), `/apply <URL>` (оценка соответствия + черновик CV и письма), `/interview`. Фишки: один агент драфтит, второй независимо ресёрчит компанию и критикует; PDF-верификация (компилирует и визуально проверяет вёрстку резюме); ATS-проверка по тексту из PDF; при переполнении режет строки с наименьшей релевантностью.

**Подвох.** Изначально заточен под датские job-порталы (есть `/add-portal` и страно-агностичные инструменты — LinkedIn/freehire); нужны Python 3.10+, Bun, LaTeX и Claude Code CLI; это каркас под свою настройку, а не «запусти-и-работает».

`case-study · subagents · slash-commands` · Источник: [github/MadsLorentzen](https://github.com/MadsLorentzen/ai-job-search)
