## 1. Claude Code 2.1.202–2.1.207: /checkup, размер динамических воркфлоу и встроенный браузер

**Зачем.** Тем, кто держит Claude Code обновлённым и хочет понимать, что реально поменялось за неделю (6–11 июля), а не читать маркетинг: новые команды обслуживания, контроль над числом субагентов и поведение auto-mode.

**Как.** Ключевое из 2.1.202–2.1.207: `/doctor` теперь полноценный чек-ап окружения с алиасом `/checkup` (чистит неиспользуемые скиллы/MCP, предлагает ужать CLAUDE.md, обновляет CLI); в `/config` появилась настройка «Dynamic workflow size» (small/medium/large — сколько агентов разворачивать в динамических воркфлоу); auto-mode доступен без флага `CLAUDE_CODE_ENABLE_AUTO_MODE` на Bedrock/Vertex/Foundry (выключить — `disableAutoMode`); добавлены встроенный десктоп-браузер и предупреждение об истечении логина; починили фриз терминала на длинных ответах. Последняя версия — 2.1.207 (11 июля).

**Подвох.** Настройка размера воркфлоу — «advisory», а не жёсткий лимит: агентов может развернуться больше. Auto-mode по умолчанию удобен, но на CI/облаке проверьте, что он не делает лишнего — при необходимости `disableAutoMode`.

## 2. Claude Code шлёт ~33k токенов до вашего промпта — у OpenCode 7k

**Зачем.** Тем, кто упирается в контекст/лимиты и хочет понимать, куда уходят токены ещё до первого вашего слова и почему «пустая» сессия уже дорогая.

**Как.** Авторы вклинили логирующий прокси между харнесом и API и померили полезную нагрузку. На задаче «ответь OK» Claude Code послал ~33k токенов против ~7k у OpenCode (4.7× на Sonnet 4.5, 3.3× на Fable 5). Разбивка: системный промпт 27 344 симв. против 9 324; 27 инструментов на 99 778 симв. против 10 на 20 856; плюс три вставных блока-напоминания с каталогом агентов. Вывод: логируйте на границе API, чтобы видеть реальный оверхед.

**Подвох.** Готового «фикса» нет — это измерение, а не рецепт: часть нагрузки (схемы инструментов) неизбежна. Цифры сняты для конкретной версии; в новых релизах оверхед меняется.

`cost-tokens · tips` · Источник: [blog/systima](https://systima.ai/blog/claude-code-vs-opencode-token-overhead)

## 3. amux: пульт для десятков параллельных агентов Claude Code / Codex / Gemini

**Зачем.** Тем, кто гоняет несколько агентов сразу и устал от того, что сессия падает ночью по переполнению контекста, агенты дублируют работу, а с телефона не подрулить.

**Как.** `git clone https://github.com/mixpeek/amux && cd amux && ./install.sh`, затем `amux register myproject --dir ~/Dev/myproject --yolo`, `amux start myproject`, `amux serve` (→ https://localhost:8822). Нужны Python 3.10+, tmux 3.2+ и хотя бы один агент-CLI. Внутри: watchdog авто-компактит и перезапускает упавшую сессию, kanban на SQLite с атомарным захватом задач (нет дублей), каналы для общения агентов через @mentions, PWA/iOS-приложение, встроенные заметки/CRM/почта/cron.

**Подвох.** Своей аутентификации нет: туннель «непубличный только по неугадываемому URL» — полагайтесь на Tailscale/localhost. Стриминг через туннель не проксируется (дашборд опрашивает раз в ~2 мин). Лицензия MIT + Commons Clause (перепродажа — по отдельной лицензии).

`automation · workflows` · Источник: [github/mixpeek](https://github.com/mixpeek/amux)

## 4. fable-baton: Fable 5 как дирижёр, дешёвые агенты как исполнители

**Зачем.** Тем, у кого быстро тает квота Fable/Opus: идея — тратить дорогую модель только на решения, а рутину делегировать дешёвым.

**Как.** `/plugin marketplace add realgarit/fable-baton`, затем `/plugin install fable-baton@fable-baton`, потом попросить «run baton-setup». Хук SessionStart вставляет правила оркестрации, UserPromptSubmit напоминает делегировать, а PostToolUse-счётчик срабатывает после 4 инлайновых вызовов подряд без делегирования. Маршрутизация: Haiku — discovery и простые проверки, Sonnet — имплементация и правки, Opus — сложный дебаг, Haiku — верификация результата.

**Подвох.** По замерам автора суммарная стоимость API почти не падает — экономится именно квота Fable (дольше живёт). Нужны Claude Code с поддержкой плагинов и доступ к Fable 5; прямые указания пользователя перебивают правила плагина.

`subagents · cost-tokens` · Источник: [github/realgarit](https://github.com/realgarit/fable-baton)

## 5. mARC: устанавливаемая «команда инженеров» из субагентов для Claude Code

**Зачем.** Небольшой команде/соло, кому нужна дисциплина ролей и «ревью безопасности перед мержем» без ручного найма — как переносимый шаблон процесса по любому репо.

**Как.** `/plugin marketplace add NexaDuo/mARC`, `/plugin install marc@nexaduo`, запуск `/marc:tech-lead`; привязать к репо — `/marc:init`. Роли: @techlead (спека, доска задач, диспетчеризация), @dev (код/IaC/тесты), @sre (деплой/наблюдаемость), @design (UI/UX), @sec (обязательный read-only ревью диффа перед merge), @research (внешние справки с цитатами). Стек читается из `.claude/team.toml` и настроек GitHub-проекта.

**Подвох.** Ранняя версия (v0.14.0). Работает только внутри Claude Code; польза зависит от того, насколько аккуратно вы опишете биндинги стека в team.toml — «из коробки» магии нет.

`subagents · workflows` · Источник: [github/NexaDuo](https://github.com/NexaDuo/mARC)

## 6. ai-job-search: агент на Claude Code, который готовит отклики на вакансии

**Зачем.** Фрилансеру/соискателю — как готовый пример, как из Claude Code собрать многошаговый рабочий процесс со слэш-командами и парой агентов, а не один промпт.

**Как.** Репозиторий MadsLorentzen/ai-job-search (Python 3.10+, Bun, LaTeX). `/setup` интервьюирует и разбирает CV/LinkedIn; `/scrape` ищет вакансии и ранжирует по 5 осям (навыки/опыт/культура/локация/траектория); `/apply` — архитектура «драфтер→критик»: один агент пишет резюме и письмо, второй со свежим контекстом придирчиво вычитывает; `/interview` готовит материалы к собеседованиям. Есть ATS-проверка через извлечение текста из PDF и запрет выдумывать факты.

**Подвох.** Не автосабмит: каждый отклик проходит ручной чекпоинт перед отправкой, автологина в порталы нет. Нужен LaTeX-тулчейн; качество зависит от честности вашего профиля.

`automation · slash-commands · subagents` · Источник: [blog/Mads Lorentzen](https://dev.to/max_quimby/ai-job-search-githubs-hottest-agent-applies-for-you-34nf)

**Зачем.** Тем, у кого длинные сессии быстро забивают контекст, из-за чего растёт стоимость и деградирует качество ответов.

## 9. «Court»: почему в длинных сессиях Claude Code зависают вызовы инструментов

**Зачем.** Тем, кто ловил «повисшие» tool-call'ы в Claude Code и терял время, не понимая, баг это модели или их конфигурации.

**Как.** Автор диагностировал по локальным логам сессии в `~/.claude/projects/` (JSONL): сообщение имеет `stop_reason: "tool_use"`, но блока tool_use нет — только текстовый `<invoke …>`. Обходной путь: не тянуть длинную сессию; при появлении «court» — сразу `/clear`, перенести состояние через Markdown-handoff и продолжить в свежей сессии. Простое «resume/go» даёт один рабочий вызов, но порча остаётся.

**Подвох.** Это лечение симптома, а не фикс: первопричина — на стороне генерации модели. Оригинал от 7 июня с обновлением 1 июля; поведение может отличаться между версиями.

`tips` · Источник: [blog/lilting](https://lilting.ch/en/articles/claude-code-court-tool-call-hang)

## 10. Циклы внутри Claude Code: перестаньте промптить, проектируйте петли

**Как.** Автор показывает, как формулировать работу как цикл (петлю) с условием остановки, а не как одиночный запрос, чтобы Claude Code повторял шаги, пока не достигнет результата.

**Подвох.** Видео-демо, короткий формат: конкретные конструкции — в ролике. Автономные циклы жгут токены и могут зациклиться — нужен явный критерий выхода и лимит итераций.

## 13. Launchpad-CLI: установка Claude Code на Windows/macOS за минуту

**Зачем.** Новичкам и тем, кто настраивает Claude Code на новой машине (особенно Windows) и не хочет вручную ставить зависимости и темизировать терминал.

**Как.** Скачать инсталлятор `.exe` (Windows) или `.pkg` (macOS) из релизов и запустить. В комплекте: Node.js+Git, тема терминала, живой статус-бар (модель, % контекста, токены), GUI-пикер папки с чипами выбора модели/флагов, ярлык и пункт «Открыть с…» в правом клике, профили (папка+модель+флаги+env+стартовые команды).

**Подвох.** Инсталлятор неподписан — Windows SmartScreen/антивирус могут ругаться (ложное срабатывание; код открыт, можно проверить на VirusTotal). Версия v2.9.2 — молодой проект.

## 14. Кейс: Claude Code + Archon генерирует готовые видео из одной ссылки

**Зачем.** Соло-креатору или маркетингу малого бизнеса, кто хочет поставить производство коротких видео на конвейер, не нанимая монтажёра, диктора и аниматора.

**Как.** По видео и репозиторию coleam00/hyperframes-ai-video-generation: Claude Code — «мозг» (план, сценарий, код), Archon (open-source мета-агент) оркестрирует шаги и состояние, Hyperframes рендерит графику из HTML-тегов с атрибутами таймлайна (удобно генерировать LLM), ElevenLabs даёт озвучку. На входе — ссылка/промпт, на выходе — ролик, готовый к ревью.

**Подвох.** Это демо энтузиаста: нужны платные сервисы (ElevenLabs) и связка инструментов; «за минуты» — про генерацию, а не про финальное качество под бренд. Требует настройки Archon/Hyperframes.

## 15. Кейс: печатный журнал из сохранённых статей на пайплайне Claude Code

**Зачем.** Пример для соло/малого бизнеса, как выстроить повторяемый контент-пайплайн, где ИИ берёт на себя редактуру и вёрстку, а детерминированный код — рендер.

**Как.** Статьи клиппятся Obsidian Web Clipper в vault. Claude Code оркестрирует: Python делает детерминированное (снапшот vault, парсинг клиппингов, заморозка картинок, HTML через Jinja2), WeasyPrint рендерит PDF под печать (после отказа от Node.js-варианта), Claude Design собрал типографику на свободных шрифтах, Codex рисует обложки. Claude как редактор читает все клиппинги и пишет JSON: разбивка по секциям, выбор главной истории, редакторская заметка.

**Подвох.** Это личный проект (единственный подписчик — сам автор), не готовый продукт: связка из нескольких инструментов (WeasyPrint, Jinja2, Codex) и ручная настройка шаблонов. Ценность — как схема, а не «поставил и работает».

`case-study · automation` · Источник: [blog/Matt Hodges](https://matthodges.com/posts/2026-07-12-personal-magazine-claude-codex)
