## 1. Субагенты, порождающие субагентов: как Anthropic теперь рекомендует настраивать Claude Code

**Зачем.** Тем, кто упирается в «один агент — один контекст» на сложных задачах: иерархия агентов позволяет разложить работу вглубь, не показывая в главном чате промежуточные шаги.

**Как.** С 2.1.172 субагенты умеют порождать своих субагентов. Рабочий паттерн из статьи: агент-ревьюер раздаёт по отдельному verifier'у на каждую найденную проблему (вместо самопроверки), и в основной разговор всплывает только верхнеуровневый итог. Правило: планировать разбиение ДО fan-out и держать единицы независимыми — два субагента, пишущие в один файл параллельно, устроят гонку.

**Подвох.** Есть предохранитель: цепочки капятся на пяти уровнях вглубь, и субагент на глубине 5 уже не получает инструмент Agent — бесконечная рекурсия исключена. Субагент-тяжёлые воркфлоу жгут до ~7× токенов против однопоточной сессии — это цена параллелизма.

`subagents` · Источник: [blog/XDA Developers](https://www.xda-developers.com/set-up-claude-way-anthropic-now-recommends-sub-agents/)

## 2. Лучшие MCP-серверы для Claude Code (2026): чем ставить первыми и точные команды установки

**Зачем.** Тем, кто тонет в выборе MCP-серверов: какие реально окупают контекст и что ставить первым под свою задачу (код-ревью, фронт, БД, веб-ресёрч).

**Как.** Топ с командами: GitHub — `claude mcp add --transport http github https://api.githubcopilot.com/mcp` (issues/PR/поиск кода/Actions; дорогой по контексту, но самый полезный); Context7 — `claude mcp add context7 -- npx -y @upstash/context7-mcp` (версионные доки, режет галлюцинации по API, дёшев); Playwright — `claude mcp add playwright -- npx @playwright/mcp@latest` (реальный браузер, самопроверка UI); Postgres — `claude mcp add postgres -- npx -y @modelcontextprotocol/server-postgres` (чтение схемы, лучше read-only роль); Exa — семантический веб-поиск для агентов. Рекомендация: 3–6 серверов, не больше.

**Подвох.** Каждый сервер ест контекст постоянно — «поставить всё» бьёт по бюджету токенов; статья обзорная (обновлена 1 июля), точные install-команды сверяйте с актуальными README серверов.

`mcp` · Источник: [blog/Nimbalyst](https://nimbalyst.com/blog/best-claude-code-mcp-servers/)

## 3. sense: MCP-сервер даёт Claude Code структурную карту кодовой базы (−32% токенов на бенчмарках)

**Зачем.** Тем, кто рефакторит или разбирается в большом/чужом коде: агент понимает связи символов и «радиус поражения» правки, а не читает половину репозитория заново на каждый вопрос.

**Как.** Установка: `curl -fsSL https://luuuc.github.io/sense/install.sh | sh` (или `go install github.com/luuuc/sense/cmd/sense@latest`). Затем `sense scan` (индексирует проект) и `sense setup` (сам прописывает MCP в Claude Code). Даёт 4 инструмента: `sense_graph` (кто вызывает/наследует, мёртвый код), `sense_search` (гибридный семантический поиск), `sense_blast` (blast radius + оценка риска правки), `sense_conventions` (конвенции проекта). Индекс живёт в фоне и обновляется по мере правок. Свежий релиз v1.11.23 от 13 июля.

**Подвох.** Ещё один локальный процесс-watcher; выигрыш максимален на больших кодовых базах, на маленьком проекте оверхед не окупится. Перед рискованными правками всё равно проверяйте вывод `sense_blast` глазами.

`mcp · refactoring` · Источник: [github/luuuc](https://github.com/luuuc/sense)

## 4. neuralmind: персистентная память + сжатие вывода для Claude Code («в 40–70× дешевле вопросы к коду»)

**Зачем.** Тем, кто жжёт токены на больших репозиториях: агент получает «выученную» карту кодовой базы и сжатые выводы инструментов вместо сырых простыней текста.

**Как.** `pip install neuralmind`, затем в проекте: `neuralmind build .` (строит индекс, tree-sitter для Python/TS/Go/Rust/Java/C/C++/C#/Ruby/PHP), `neuralmind install-hooks .` (ставит PostToolUse-хуки сжатия), `neuralmind watch .` (держит индекс свежим). На старте сессии автоматически грузится `SYNAPSE_MEMORY.md` с выученными связями. Заявлено: 100% recall нужных файлов при 38–85× меньше токенов, чем при вставке файлов целиком.

**Подвох.** Цифры «40–70×» — из README автора, независимых бенчмарков нет; на демо-фикстуре среднее сжатие скромнее (~5.5×). Плюс к приватности (всё локально), но добавляет в проект собственную обвязку хуков.

`cost-tokens · mcp` · Источник: [github/dfrostar](https://github.com/dfrostar/neuralmind)

## 5. Kastra: политики-guardrails для Claude Code — каждая команда и правка проходит проверку локально

**Зачем.** Тем, кто пускает агента в auto-режиме и боится деструктива: жёсткие правила уровня «нельзя трогать .env.production / rm -rf», проверяемые ДО выполнения, а не по доброй воле модели.

**Как.** Установка Edge-модуля: `brew install kastra-labs/tap/kastra-edge` (или macOS-приложение с сайта). Демон локально перехватывает shell-команды, правки файлов и API-вызовы Claude Code/Codex/Cursor и выносит решение за <1мс. Политики «типизированные, версионируемые, ревьюятся как код». Примеры из сайта: DENY `rm -rf node_modules` (destructive.shell), DENY запись в `.env.production` (secrets.protect), ALLOW скоуп-ограниченные чтения БД.

**Подвох.** Проприетарный продукт (не open-source); «бесплатно навсегда» — только для соло-разработчика, командные/enterprise-фичи платные; это ещё один фоновый демон с доступом ко всему, что делает агент, — доверие к вендору обязательно.

`automation · tips` · Источник: [blog/Kastra](https://kastra.ai/)

## 6. Power-user советы по Claude Code от Anthropic: worktree-параллелизм, /effort, /voice, авто-память

**Зачем.** Тем, кто уже освоил базу и хочет выжать из Claude Code скорость: официальные приёмы «как в команде Anthropic», а не догадки из блогов.

**Как.** Параллелизм: `claude --worktree` (или `--worktree my_worktree`, опц. `--tmux`) — несколько сессий, каждая в своём git-worktree; `/batch` — крупные миграции десятками агентов. Планирование: Shift+Tab → plan mode; `/effort` (low/medium/high/xhigh/max/auto). Верификация (у них — приём №1): Chrome-расширение для фронта, `/simplify` в конце промпта — параллельное ревью качества. Память: общий `CLAUDE.md` в git + `/memory` для авто-памяти; `@claude` в комментарии PR обновляет CLAUDE.md. Кастомизация: `/color` — раскрасить параллельные сессии, `/voice` — голосовой ввод.

**Подвох.** Официальная страница-эвергрин («обновлено более 2 недель назад» — точную дату Anthropic не показывает); часть команд (`/batch`, `/voice`) может зависеть от плана и версии CLI.

`tips` · Источник: [docs/Anthropic](https://support.claude.com/en/articles/14554000-claude-code-power-user-tips)

## 7. CodexBar: лимиты и расход Claude Code (и Codex) в меню-баре macOS — без отдельного логина

**Зачем.** Тем, кто упирается в недельные лимиты Claude/Codex и хочет видеть остаток, не открывая дашборды: индикатор прямо в меню-баре.

**Как.** macOS 14+: `brew install --cask codexbar` (или скачать с GitHub Releases); есть tarball-CLI `codexbar` для скриптов/CI. Читает использование из уже существующих источников — OAuth-сессии, cookies браузера, конфиги провайдеров, локальные логи — отдельный пароль не нужен (достаточно быть залогиненным в самих CLI/браузере). Свежая версия 0.42.1 от 12 июля.

**Подвох.** Меню-бар — только macOS (CLI работает и на Linux); заявлен «privacy-first», но по сути читает cookies и локальные файлы провайдеров — доверяйте на свой риск. Точность метрик ограничена тем, что провайдер отдаёт наружу.

`cost-tokens` · Источник: [github/steipete](https://github.com/steipete/CodexBar)

## 8. sessions: полнотекстовый поиск и память по всем сессиям Claude Code / Codex / Pi

**Зачем.** Тем, кто теряет наработки между сессиями и инструментами: «где я это уже делал?» — поиск по всей истории Claude Code, Codex и Pi в одном месте.

**Как.** `brew install nicknisi/formulae/sessions` (или из исходников на Bun). CLI: `sessions` (браузер сессий с fuzzy-выбором), `sessions <query>` (полнотекстовый поиск), `sessions --here` (только текущий репозиторий), `sessions context` (контекст-праймер для репо), `sessions report` (дашборд расхода в браузере). После `sessions setup` — регистрируется как MCP-сервер для Claude Code/Cursor и даёт агенту поиск и выжимки истории. Свежий релиз 1.13.0 от 13 июля.

**Подвох.** Ранний проект (14 релизов); индексирует локальные транскрипты — польза растёт вместе с накопленной историей, «с нуля» выигрыша нет.

`workflows · mcp` · Источник: [github/nicknisi](https://github.com/nicknisi/sessions)

## 9. superset: редактор кода для «армии» агентов — Claude Code параллельно, каждый в своём git-worktree

**Зачем.** Тем, кто гоняет несколько агентов параллельно и хочет удобный GUI вместо зоопарка терминалов: изоляция задач через worktree исключает гонки за файлы, а diff-вьюер держит правки под контролем.

**Как.** Установка из исходников: `git clone https://github.com/superset-sh/superset.git`, `cd superset`, `./.superset/setup.local.sh`, затем `bun run dev`. Работает с любым CLI-агентом, живущим в терминале; каждая задача получает отдельный git-worktree, есть встроенный просмотр диффов и терминал. Есть готовые десктоп-релизы (v1.15.0 от 13 июля).

**Подвох.** Лицензия Elastic License 2.0 (source-available, НЕ классический open-source — есть ограничения на коммерческое использование как сервиса); проект активный, но это «редактор эпохи агентов» на раннем этапе — под одиночную сессию избыточен.

`ide-integration · workflows` · Источник: [github/superset-sh](https://github.com/superset-sh/superset)

## 10. cmuxLayer: MCP-оркестратор — чтобы агенты Claude Code «видели» терминалы друг друга

**Зачем.** Тем, кто строит мульти-агентные пайплайны и упирается в изоляцию агентов: cmuxLayer даёт им общий «пульт» — видеть и дёргать друг друга программно, а не через хрупкие обёртки.

**Как.** `brew install etanhey/layers/cmuxlayer` (нужен запущенный экземпляр cmux). Сервер даёт 42 инструмента; в палитре по умолчанию: `spawn_agent`, `send_to`, `wait_for`, `read_screen`, `broadcast`, `dispatch_to_agent`, `stop_agent` и др. Умеет мультиплексировать терминал (сплиты, воркспейсы), читать метаданные агентов пяти форматов и слать задачи через инбоксы. Unix-сокет вместо CLI-подпроцесса даёт задержку ~0.1мс против ~142мс.

**Подвох.** Очень ранний проект (17 звёзд), завязан на cmux как рантайм; для одной-двух сессий этот слой избыточен — окупается только на реальной оркестрации нескольких агентов.

`automation · mcp` · Источник: [github/EtanHey](https://github.com/EtanHey/cmuxlayer)

## 11. agentic-awesome-skills: библиотека из 1948+ скиллов, ставится в Claude Code одной командой

**Зачем.** Тем, кто не хочет писать скиллы с нуля: большой каталог готовых плейбуков, которые можно ставить точечно и подсматривать как образцы структуры SKILL.md.

Внутри — 1948+ скиллов, бандлы, воркфлоу и плагины; лицензии MIT / CC BY 4.0, есть ledger атрибуции 100+ официальных и community-источников. **Подвох.** Это community-проект, НЕ официальный Anthropic; при 1900+ скиллах качество разнородно — ставьте нужное точечно, а не всё сразу (иначе распухнут контекст и токены). Перед установкой скилла имеет смысл заглянуть в его SKILL.md.

## 13. 9lives: самозаживляющиеся тесты — Claude Code сломал селектор, `9l heal` его чинит

**Зачем.** Тем, у кого агент правит фронт и постоянно роняет e2e-тесты на съехавших селекторах: вместо ручной починки — авто-ремонт с показом диффа.

**Как.** Установка: `curl -sL 9lives.run | sh` или `pip install 9lives` (нужен Node.js ≥18). Tier 1 (офлайн, без LLM) чинит селекторы по text/testid/id/class/aria-label из снапшота падения; Tier 2 роутит сложные случаи в ваш существующий CLI-агент (Claude Code/Codex/OpenCode) или по API-ключу. Всегда показывает дифф и требует подтверждения, если не передан `--yes`.

**Подвох.** Совсем ранний (v0.1.3 от 9 июля, 11 коммитов, один автор) — на прод пока с осторожностью; чинит симптом (селектор), а не причину (почему UI изменился) — легко замаскировать реальную регрессию, если не смотреть диффы.

`testing` · Источник: [github/Quality-Max](https://github.com/Quality-Max/9lives)
