## 1. Claude Code 2.1.208–2.1.210: режим скринридера, vim-ремапы и починка изоляции worktree

**Зачем.** Тем, кто гоняет параллельных агентов: до 2.1.210 сабагент с `isolation: 'worktree'` мог выполнять git-мутации против основного репозитория, а не своей копии — это закрыли. Плюс доступность: полноценный режим скринридера.

**Как.** Обновиться: `claude update`. Скринридер: `claude --ax-screen-reader` или env `CLAUDE_AX_SCREEN_READER=1`. Vim-ремапы: настройка `vimInsertModeRemaps` (например, `jj` → Escape). Корпоративные лаунчеры: `CLAUDE_CODE_PROCESS_WRAPPER`. `/usage` при рейт-лимите теперь показывает последние известные бары с пометкой «as of», а не ошибку.

**Подвох.** В 2.1.210 — 33 изменения, но в основном стабилизация (фоновые воркеры, кеш плагинов, MCP); громких новых фич на этой неделе нет.

`tips · ide-integration · automation` · Источник: [docs/Anthropic](https://code.claude.com/docs/en/changelog#2-1-210)

## 3. vigiles — «Lighthouse» для вашего CLAUDE.md: проверяет, что скиллы и хуки реально работают

**Зачем.** У кого конфиг оброс инструкциями и скиллами: половина может молча не работать — мёртвые пути к скриптам, коллизии описаний скиллов, противоречащие правила. vigiles находит это за минуту, локально.

**Как.** `npx vigiles audit` — отчёт с оценкой A–F. Дальше `vigiles init` → `lint` (гейт для CI), `test` (поведенческие тесты харнесса), `eval` (замер скиллов). Пять категорий: правдивость ссылок на тулзы/пути, срабатывание скиллов, структура, безопасность комбинаций прав, покрытие тестами.

**Подвох.** Свежий (v14.2.0 от 15.07); поведенческие тесты и eval гоняют модель — тратят токены; оценка A–F — эвристика.

`testing · skills · hooks · tips` · Источник: [github/zernie](https://github.com/zernie/vigiles)

## 4. claude-architect: сессия Fable/Opus проектирует, дешёвые исполнители пишут код

**Зачем.** Экономить дорогие токены топ-модели: Fable/Opus остаётся на требованиях, декомпозиции и ревью, а рутинную имплементацию выполняет дешёвый «лейн» — Codex, OpenCode-провайдеры, локальные модели (Pi, нулевая цена токена) или свой headless-агент.

**Как.** `claude plugin marketplace add Pythoughts-labs/claude-architect` → `claude plugin install claude-architect@claude-architect`. Затем `/claude-architect:delegate <задача>` — можно явно назвать лейн. Архитектор пишет спеку, смотрит реальный дифф и решает — принять или нет.

**Подвох.** Write-confinement для Codex-лейна сертифицирован только на macOS arm64; Node.js 22+; таймауты 900 с; параллельные записи в одно дерево запрещены. v0.6.0 — ранняя.

`subagents · workflows · cost-tokens` · Источник: [github/Pythoughts-labs](https://github.com/Pythoughts-labs/claude-architect)

## 5. Shipwright — автономный конвейер поставки для Claude Code: план → код → ревью → деплой

**Зачем.** Командам, которые хотят автономную доставку фич, но не готовы отдавать код во внешние сервисы: MIT-лицензия, деплой в своей инфраструктуре, локальный дашборд метрик.

**Как.** `/plugin install shipwright@app-vitals/shipwright`. Пайплайн: `/shipwright:prd` → `/shipwright:plan-session` → `/shipwright:dev-task` → `/shipwright:review` → `/shipwright:deploy`. Для Kubernetes есть Helm-чарт; уведомления в Slack; общая очередь задач на несколько агентов.

**Подвох.** Полный стек тяжёлый: Bun, go-task, Docker, PostgreSQL, tmux. Навязывает свою «четырёхслойную» тест-архитектуру; дашборд метрик по умолчанию офлайн (фикстуры).

`automation · workflows · ci-cd` · Источник: [github/app-vitals](https://github.com/app-vitals/shipwright)

## 6. AgentWire — клавиатурный «пульт» для флота Claude Code-агентов в git-worktree

**Зачем.** Тем, кто уже гоняет 3+ параллельных агента и утонул в tmux-окнах: один интерфейс со всем флотом — сетка сессий как Mission Control (F3), переключение по Tab, палитра Cmd/Ctrl+K.

**Как.** `uv tool install agentwire-dev` → `agentwire init` → `agentwire portal start` → `http://127.0.0.1:8765` в Chrome. Нужны Python 3.10+, tmux, Claude Code. Паттерны: Scheduler (автономные регулярные задачи с гейтами), Council (промпт веером на роли «мозг/совесть/критик»), Briefing. Голос: Kokoro-82M TTS на CPU.

**Подвох.** Только локальная машина (LAN — через SSL), без tmux не работает; блок-лист ~312 опасных паттернов не отменяет присмотра.

`subagents · automation · workflows` · Источник: [github/dotdevdotdev](https://github.com/dotdevdotdev/agentwire-dev)

## 8. cc-hindsight: превращает историю Claude Code в библиотеку «правильных первых промптов»

**Зачем.** Если вы раз за разом объясняете Claude одно и то же: инструмент вытаскивает паттерны из реальной истории и делает заготовки — «промпт, который вы бы написали, зная то, что знаете теперь».

**Как.** `npx cc-hindsight distill`: scan (инвентаризация ~/.claude/projects) → export (markdown, без LLM) → distill (Claude API кластеризует и пишет улучшенные промпты, с явного согласия). Далее `list`, `copy <slug>`, `preferences` (блок для CLAUDE.md).

**Подвох.** Windows не поддерживается; distill стоит токенов; сессии >400k символов отбрасываются; Node 22+.

## 10. Alluvia: локальный анализ всей вашей истории Claude Code, Cursor и ChatGPT

**Зачем.** В истории агентов закопаны решения, которые вы забыли: Alluvia показывает, где вы решали одно и то же дважды в разных инструментах, и какие темы не довели до конца.

**Как.** `uv tool install alluvia` → `alluvia init` → `alluvia refresh` (дистилляция → эмбеддинги → кластеризация → связи) → `alluvia serve --open` (дашборд на localhost:8177). Эмбеддинги локальные (ONNX/fastembed) — сырые сессии никуда не отправляются.

**Подвох.** ChatGPT — только ручной ZIP-экспорт; Windsurf/Antigravity пропускаются; «предложения» генерирует LLM — фильтровать головой. v0.3.0.

## 11. Trollbridge — умный HTTP-прокси: агент работает свободно, но каждый выход в сеть — по политике

**Зачем.** Запускать Claude Code с ослабленными permission'ами без страха неконтролируемых обращений в сеть: «гейтим провод, а не файловую систему».

**Как.** `curl -fsSL https://trollbridge.dev/install.sh | sh` → `trollbridge quickstart` → выставить `HTTP_PROXY`/`HTTPS_PROXY`. Три слоя: статические allow/deny-списки → опциональный LLM-советник для неоднозначных запросов → ручное одобрение одной клавишей.

**Подвох.** Не гейтит файлы и процессы — авторы советуют пару с песочницей (VM/контейнер/WSL2). Установка curl|sh — прочитайте скрипт.

`automation · tips` · Источник: [hn](https://trollbridge.dev/)

## 12. Пять рабочих хуков Claude Code: автоформат, блок опасных правок, нотификации, direnv и подмена ввода

**Зачем.** Хуки срабатывают детерминированно — в отличие от инструкций в промпте. Пять примеров закрывают самые частые бытовые нужды без чужих плагинов.

**Как.** PostToolUse (`Edit|Write`): `jq -r '.tool_input.file_path' | xargs npx prettier --write`. PreToolUse с exit 2 — блок правок .env и force push. Notification + `osascript` — десктоп-уведомления. CwdChanged + `direnv export bash >> "$CLAUDE_ENV_FILE"`. PreToolUse с `updatedInput` — подмена команды до исполнения.

**Подвох.** Примеры под macOS/Unix (osascript, jq, direnv) — на Windows адаптировать. Статья от 30.06 — на границе окна свежести.

## 14. Кейс GoodBarber: production-MCP-сервер на 150+ инструментов — агенты управляют приложениями клиентов

**Зачем.** Образец для небольшого SaaS, как сделать продукт «agent-ready» один раз (через MCP), вместо интеграции под каждого ассистента. Клиенты GoodBarber — малый бизнес, ведущий приложения без разработчиков.

**Как.** 150+ типизированных тулзов по доменам (`cms_*`, `shop_*`), OAuth 2.0 на конкретное приложение клиента. Главные решения: каждая запись возвращает `verification_required: true` — агент перечитывает созданный объект (страховка от «галлюцинированного успеха»); фичи-гейты прячут недоступные тулзы; JSON-схемы валидируют аргументы до исполнения.

**Подвох.** Рецепт для продуктов, у которых уже есть API и бэк-офис. Дисциплина read-back — ручная работа команды: сам MCP этого не навязывает.

`case-study · mcp · automation` · Источник: [blog/Pierre-Laurent Medori (GoodBarber)](https://dev.to/pierrelaurentmedori/building-a-production-mcp-server-how-we-made-goodbarber-agent-ready-without-the-glue-code-4co6)

## 15. Видео-кейс: «маркетинговая команда из 47 агентов за $0» на Claude Code и одном GitHub-репо

**Зачем.** Соло-предпринимателю или маленькой команде без бюджета на маркетологов: упаковать маркетинговую рутину (посты, позиционирование, разбор аудитории) в набор специализированных сабагентов.

**Как.** По ролику: клонируется один GitHub-репозиторий с готовыми определениями агентов-«ролей», Claude Code подхватывает их как сабагентов и раздаёт задачи. За конкретикой — сам ролик.

**Подвох.** Описание видео WebFetch не отдаёт — репозиторий и качество агентов не проверены. «За $0» — кликбейт: подписка/токены всё равно стоят денег; 47 агентов — это 47 промпт-файлов, а не 47 сотрудников.

`case-study · subagents · automation` · Источник: [youtube/Hyperautomation Labs](https://www.youtube.com/watch?v=S48lbJRwmY8)
