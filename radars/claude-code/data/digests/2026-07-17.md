## 1. Claude Code 2.1.211: стриминг текста сабагентов и права, общие для всех worktree

**Зачем.** Тем, кто гоняет Claude Code в скриптах и CI через `--output-format stream-json` (раньше текст сабагентов в вывод не попадал), и тем, кто работает в git worktree и устал заново подтверждать одни и те же разрешения в каждом дереве.

**Как.** Обновиться (`claude update`). Стриминг сабагентов: `claude -p --output-format stream-json --forward-subagent-text` или переменная `CLAUDE_CODE_FORWARD_SUBAGENT_TEXT`. Правила «always allow» теперь пишутся в корень репозитория и переживают сессии и worktree. Заодно усилена защита: превью разрешений нейтрализует невидимые Unicode-символы (bidi, zero-width), а решение `ask` из PreToolUse-хука больше не перекрывается auto-режимом.

**Подвох.** Релиз почти целиком из ~40 фиксов. Самый денежный: на Bedrock/Vertex/Mantle/Foundry регрессия prompt-caching биллила хвостовой системный блок как свежие входные токены при каждом запросе — если счета внезапно выросли, обновляться обязательно.

`subagents · cost-tokens · tips` · Источник: [docs/Anthropic](https://code.claude.com/docs/en/changelog#2-1-211)

## 2. Разбор 241-ходовой сессии Claude: как ловить моменты, где агент вас не слушает

**Зачем.** Тем, у кого длинные сессии Claude Code «в целом ок», но время утекает непонятно куда: разбор показывает, как превратить повторяющиеся сбои агента в правила для следующих сессий.

**Как.** Автор оценивает сессию (30 часов, 7,5 млн токенов, 4038 вызовов инструментов) инструментом Capacitor по трём измерениям: Plan Adherence 4/5, Quality 5/5, Efficiency 3/5. Инструмент указывает номера ходов, где потерялась эффективность, и генерирует рекомендации вида «проверяй поведенческие допущения по реальным транскриптам до реализации». Практические правила: требовать немедленной правки замечаний ревью (агент склонен откладывать) и не верить на слово утверждениям о документации.

**Подвох.** Шестифазная фича на непроверенном допущении потребовала полного редизайна после 40+ ходов — оценка постфактум не заменяет проверку допущений в начале. Capacitor — продукт самого автора (Kurrent), статья отчасти витрина.

`prompting · workflows · tips` · Источник: [blog/Tony Young](https://www.kurrent.io/blog/when-your-coding-agent-doesnt-listen/)

## 3. Fuse: MCP-сервер с Roslyn — Claude Code проверяет C#-правки компилятором до записи

**Зачем.** Командам с C#/.NET-кодовой базой, где Claude Code «промахивается» по типам и DI-регистрациям и ломает компиляцию после серии правок.

**Как.** `dotnet tool install -g Fuse`, затем `fuse mcp install --rules` — конфиг MCP-клиента прописывается автоматически. Ключевые инструменты: `fuse_check` — типопроверка правки без изменения рабочего дерева, `fuse_find` — интерфейс → реализация, `fuse_impact` — все вызывающие перед рефакторингом, `fuse_test` — только релевантные тесты. Индекс живёт в `.fuse/fuse.db` и переживает ходы агента. Заявленная экономия: 38–44% лишних токенов контекста.

**Подвох.** Семантика только для .NET (для прочих языков — синтаксический фолбэк), нужен git-репозиторий, локально крутится постоянный демон. Проект свежий, цифры экономии — от автора.

`mcp · cost-tokens · testing` · Источник: [github/Litenova Solutions](https://github.com/Litenova-Solutions/Fuse)

## 4. Маркетплейс Power BI-плагинов для Claude Code: скиллы DAX/TMDL, агенты и хуки-валидаторы

**Зачем.** Аналитикам и BI-разработчикам: Claude Code из коробки плохо знает специфику TMDL/PBIP-файлов и DAX — плагины дают готовую экспертизу вместо ручного объяснения в каждой сессии.

**Как.** `claude plugin marketplace add data-goblin/power-bi-agentic-development`, дальше ставить нужное по имени (например `tabular-editor@power-bi-agentic-development`). Скиллы активируются по контексту задачи (DAX, Power Query, TMDL, авторинг отчётов с Deneb/Python/R), агенты берут многошаговые задачи вроде валидации модели, хуки после каждого изменения проверяют файлы и ловят ошибки рано.

**Подвох.** На Windows нужен включённый long path support, иначе установка падает с «Filename too long». Версии 26.26–26.38 — объявленный «ломающий переход»: для стабильности авторы советуют пинить 26.25.

## 5. usage: квота, токены и стоимость Claude Code всегда на экране — без API-вызовов

**Зачем.** Тем, кто регулярно упирается в лимиты Pro/Max посреди работы: постоянный индикатор с цветовой подсветкой избавляет от сюрприза «квота кончилась» на середине задачи.

**Как.** macOS: `brew install --cask aqua5230/usage/usage`; Windows: usage-windows.zip из Releases (нужен WebView2 Runtime). Данные берутся из локальных логов — ни одного API-вызова и никаких паролей. Бонусы: HTML-отчёты с трендами и разбивкой затрат, тумблер «Token Saver», Progress Concierge — автоматически возобновляет сессии Claude Code с прежним контекстом, 10 визуальных тем.

**Подвох.** До первой сессии индикатор показывает «--»; часть панелей только на macOS. Стоимость — расчётная оценка по логам, а не данные с биллинга Anthropic.

`cost-tokens · tips` · Источник: [github/aqua5230](https://github.com/aqua5230/usage)

## 7. cc-session-recover: Claude Code сам дописывает работу, когда квота откатывается

**Зачем.** Тем, кто упирается в 5-часовые и недельные окна лимитов на длинных задачах: вместо ручной реанимации сессия дожидается квоты и продолжает сама.

**Как.** `npx cc-session-recover init /path/to/project` — ставит хуки и файлы восстановления, не трогая существующий конфиг. В сессии выполнить `/session-recover` (интервал проверки в минутах, по умолчанию 30). Механика: хук на сбой записывает ошибку в `.claude/quota-blocked.json`, HANDOFF.md фиксирует состояние задачи и следующие шаги, периодическая проверка возобновляет работу, когда окно открылось. Есть вариант с внешним watcher-процессом.

**Подвох.** Квоту не обходит — только ждёт и продолжает. Нужно держать процесс Claude Code открытым (или гонять watcher: ему нужны jq, Node.js и claude в PATH). Ошибки аутентификации и биллинга не ретраит; «ночную» работу всё равно ревьюить.

`hooks · automation · cost-tokens` · Источник: [github/softcane](https://github.com/softcane/cc-session-recover)

## 8. Почему Domain-Driven Design отлично ложится на работу с Claude

**Зачем.** Тем, у кого Claude Code на сложной бизнес-логике выдаёт «правдоподобно, но не то»: DDD-приёмы дают модель, о которой с агентом можно говорить без двусмысленностей.

**Как.** Держать в CLAUDE.md словарь домена (точные термины вроде «hold», «booking», «expire»); работать по одному bounded context за сессию; перед кодом просить модель переформулировать инварианты домена — расхождения всплывают до реализации; тесты писать как исполняемые контракты до кода; на ревью проверять соответствие доменному языку, а не только корректность.

**Подвох.** LLM по умолчанию тянет в противоположную сторону: анемичные CRUD-модели, выдуманная терминология, переусложнение паттернами и дрейф от словаря в длинных сессиях. И сам DDD стоит дозировать по реальной сложности домена.

`prompting · workflows` · Источник: [blog/Menelaos Vergis](https://menelaos.vergis.net/posts/Why-Domain-Driven-Design-Is-a-Great-Fit-for-Coding-with-Claude)

## 9. Локальное зеркало приватных репо: поиск для Claude Code за миллисекунды вместо API GitHub

**Зачем.** Командам с десятками приватных репозиториев: вопрос «в каком репо у нас X?» через Claude Code становится мгновенным, а не археологией через веб-поиск GitHub.

**Как.** Bash-скрипт ~60 строк (private-github-search-sync.sh) shallow-клонирует и синхронизирует все репо (~2,8 ГБ на 228 штук); поверх — скилл Claude Code, чтобы спрашивать естественным языком («which repo has X?»). Цифры автора: первый поиск ~1 с, из кэша — 11 мс, проверка синка по всем репо ~15 с. Причина: gh/REST API ищет только default-ветку, файлы до 384 КБ и репо с активностью за год.

**Подвох.** Зеркалятся только default-ветки и без форков (тот же охват, что у веб-поиска GitHub), зато включая архивные репо и без задержки индексации. Нужны место на диске и регулярный sync.

`tips · skills` · Источник: [blog/ykdojo](https://ykdojo.github.io/claude-code-tips/content/the-missing-private-github-search.html)

## 10. Claude Code может насыщать вашу машину: осиротевшие busy-loop процессы живут днями

**Зачем.** Всем, у кого после активной работы с агентом ноутбук греется «на ровном месте»: фоновые процессы, запущенные инструментом Bash, могут пережить и краш, и отмену сессии.

**Как.** Диагностика: `uptime` (load против числа ядер), затем `ps -Ao pcpu,pid,ppid,user,comm -r` — искать процессы с PPID 1 и большим etime, детали через `ps -o pid,ppid,lstart,etime,pcpu,args -p <pid>`. Защита в скриптах, которые пишет агент: `trap 'kill $LOADPIDS 2>/dev/null' EXIT INT TERM`; в неинтерактивных шеллах `jobs -p` возвращает пустоту — PID собирать через `$!`.

**Подвох.** Обычного `kill` достаточно, `-9` не нужен. Load average запаздывает по определению — об успехе чистки судить по списку процессов, а не по метрике.

`tips` · Источник: [blog/Sidhant Panda](https://dev.to/sidhantpanda/claude-might-be-saturating-your-machine-3h07)

## 11. Миграция с Webflow: честный разбор «худшего способа использовать Claude Code»

**Зачем.** Тем, кто собирается доверить агенту точное воспроизведение дизайна: это редкий детальный разбор неудачи — видно, где именно подход ломается.

**Как.** Схема была: перестраивать страницы маршрут за маршрутом, а через Playwright-интеграцию снимать скриншоты и итерироваться до совпадения с макетом. По факту: 3+ минуты на скриншот, циклы застревали, первые проходы галлюцинировали шрифты, размеры и радиусы скруглений. Заметно лучше стало, когда Claude получил доступ к готовым компонентам. Вывод автора: скаффолдинг и бэкенд — агенту, дизайн-детали — руками.

**Подвох.** Один проект и одна модель (Sonnet 4.6) — не приговор подходу: с жёсткой дизайн-системой и библиотекой компонентов результаты у автора же улучшались по ходу работы.

`refactoring · tips · case-study` · Источник: [blog/sisyphus.bar](https://www.sisyphus.bar/blog/migrating-off-webflow)

## 15. Как я быстро выросла в соцсетях, связав Claude и Blotato

**Зачем.** Солопренёрам и малому бизнесу без отдельного человека на соцсети: контент-конвейер «генерация → адаптация под площадки → автопостинг по расписанию».

**Как.** Видео (15.07.2026): рабочий процесс автора — Claude пишет и адаптирует посты под разные площадки, Blotato публикует по расписанию в несколько сетей разом. Канал Sabrina Ramonov специализируется на таких no-code автоматизациях.

**Подвох.** Blotato — платный сервис, и автор тесно с ним аффилирована (продвигает его во всех своих видео) — это отчасти витрина продукта. Описание видео программно вытащить не удалось — цифры роста только в ролике.
