## 1. wshobson/agents: маркетплейс из 94 плагинов (203 агента, 175 скиллов) для Claude Code

**Зачем.** Тем, кто не хочет писать субагентов с нуля, а хочет взять проверенный набор специалистов под конкретный домен.

**Как.** `/plugin marketplace add wshobson/agents`, затем `/plugin install python-development` (или любой из 94 плагинов). Внутри — многоуровневая стратегия моделей: Opus на архитектуру, Sonnet на доки, Haiku на рутину.

**Подвох.** Это сторонний каталог, качество плагинов разное; много подключённых ролей = раздувание контекста и токенов, ставьте точечно.

`subagents · skills · slash-commands` · Источник: [github/wshobson](https://github.com/wshobson/agents)

## 2. claude-code-studio: браузерный воркспейс над Claude Code CLI (kanban, планировщик, Telegram, SSH)

**Зачем.** Тем, кто гоняет несколько задач разом и хочет пульт (доска + расписание + удалёнка) вместо голого терминала.

**Как.** Быстрый старт: `npx github:Lexus2016/claude-code-studio`, открыть `http://localhost:3000`, задать пароль. Есть Docker (`docker compose up -d --build`) и десктоп-сборки. Нужен Node 18+ и установленный Claude Code CLI.

**Подвох.** Десктоп-версия single-user и привязана к 127.0.0.1; планировщик — до 5 параллельных воркеров; известный баг npm 10.x с git-install (обновить npm до последней).

`workflows · automation · ide-integration · mcp` · Источник: [github/Lexus2016](https://github.com/Lexus2016/claude-code-studio)

## 3. CodeAlive-AI/ai-driven-development: 22 скилла и safety-hook bash-guard для Claude Code

**Зачем.** Тем, кто хочет безопаснее пускать агента в bash и разом получить базовый набор инженерных скиллов.

**Как.** `npx skills add CodeAlive-AI/ai-driven-development` (все скиллы) или `--skill prompt-engineering` (один); либо `/plugin marketplace add CodeAlive-AI/ai-driven-development` + install. Хук ставится скриптом `install-prebuilt.sh`.

**Подвох.** Подход «ask, не deny»: bash-guard требует подтверждения, а не предотвращает действие; кросс-платформенность в ущерб специфичным фишкам агента; активные скиллы всё равно едят токены.

`skills · hooks` · Источник: [github/CodeAlive-AI](https://github.com/CodeAlive-AI/ai-driven-development)

## 4. deja-vu: слой памяти для Claude Code — семантический recall по прошлым сессиям через MCP

**Зачем.** Если постоянно ловите «где-то я это уже чинил»: вытащить прошлое решение без ручного грепа по логам.

**Как.** Установка `curl -fsSL https://raw.githubusercontent.com/vshulcz/deja-vu/main/install.sh | sh` или `brew install vshulcz/tap/deja-vu`. Подключение: `deja install --all` (MCP-инструмент recall) или `deja install --auto` (авто-подмешивание на SessionStart, ≤2 КБ). Инструменты: recall / recall_context / remember.

**Подвох.** По умолчанию поиск лексический, семантика требует Ollama/LM Studio; фильтра по проектам пока нет; Windows-сборки есть, но обкатаны macOS/Linux; транскрипты субагентов по умолчанию пропускаются.

`mcp · cost-tokens` · Источник: [github/vshulcz](https://github.com/vshulcz/deja-vu)

## 5. pluxx: один исходник плагина → нативные плагины для Claude Code, Cursor, Codex и OpenCode

**Зачем.** Авторам плагинов и MCP-серверов, кто устал синхронизировать четыре реализации вручную.

**Как.** `npx @orchid-labs/pluxx init --from-mcp <url> --name my-plugin --yes`. Есть `autopilot` для полностью автоматического пути и вариант для локального MCP: `--from-mcp "npx -y -p @acme/mcp acme-mcp"`. Нужен Node ≥18.

**Подвох.** Заточен под 4 хоста, остальные цели помечены как beta; работает только с уже настроенными MCP; общий рантайм требует детерминированного bootstrap для тяжёлых зависимостей.

`mcp · automation · skills` · Источник: [github/orchidautomation](https://github.com/orchidautomation/pluxx)

## 6. CrewCtl: оркестратор-«оператор» для команды из Claude Code, Codex, Gemini и OpenCode

**Зачем.** Тем, кто параллелит несколько CLI-агентов и хочет единый пульт с планированием и ревью вместо ручного жонглирования терминалами.

**Как.** Запуск без установки: `npx @omerrgocmen/crewctl` (или глобально `npm install -g @omerrgocmen/crewctl && crewctl`), интерфейс на `http://localhost:4317`. Есть Kanban (Pending·Running·Done·Failed), пресеты расписаний, live-diff и авто-версии с откатом в один клик. Node 18+, без рантайм-зависимостей.

**Подвох.** Расписания работают только пока «мотор» запущен (без unattended-автозапуска); откат версий — только на простое; для критичной работы автор советует обычный git. Проект молодой.

`workflows · subagents · automation` · Источник: [github/omergocmen](https://github.com/omergocmen/CrewCtl)

## 7. quil: tmux-альтернатива, которая переживает ребут и авто-возобновляет сессии Claude Code

**Зачем.** Тем, кто теряет длинные сессии Claude Code после ребута или сна машины, и тем, кто хочет, чтобы агент сам рулил панелями терминала.

**Как.** Установка: `curl -sSfL https://raw.githubusercontent.com/artyomsv/quil/master/scripts/install.sh | sh`, запуск `quil`. Для управления панелями добавить MCP-сервер в конфиг Claude Code: `"quil": {"command": "quil", "args": ["mcp"]}` — откроет 15+ инструментов (list_panes, read_pane_output, send_to_pane). После ребута каждая панель переподключается к прежнему диалогу сама.

**Подвох.** Проект свежий (v1.37.0), явных ограничений в README нет — обкатывать на некритичном; Windows-сборка ставится вручную из релизов.

`workflows · automation · mcp` · Источник: [github/artyomsv](https://github.com/artyomsv/quil)

## 8. symphony-rust: десктоп-оркестратор, который сажает Claude Code на задачи из доски Linear

**Зачем.** Небольшой команде или соло-мейкеру с потоком задач в Linear, кто хочет «автономную инженерную бригаду».

**Как.** Symphony мониторит доску Linear, диспатчит каждую issue в отдельный клон репозитория на агента (Claude Code/Codex) и собирает результаты обратно в контекст ведущего агента; вы триажите и ревьюите.

**Подвох.** Очень свежий проект (Rust, мало звёзд), заточен под связку с Linear; автономный прогон агентов требует присмотра и лимитов на расход.

`workflows · automation` · Источник: [github/anantjain-xyz](https://github.com/anantjain-xyz/symphony-rust)

## 9. nix-ai: весь сетап Claude Code (плагины, хуки, агенты, 15+ MCP) декларативно в одном Nix-флейке

**Зачем.** Nix/home-manager-пользователям, кто хочет воспроизводимый версионируемый сетап Claude Code вместо ручной возни с конфигами на каждой машине.

**Как.** Добавить в inputs флейка `url = "github:dryvist/nix-ai"` и подключить модуль `nix-ai.homeManagerModules.default`. Опции: `programs.claude.model`, `programs.claude.effortLevel`, `programs.claude.trustedProjectDirs`; в комплекте cclint, doppler-mcp, claude-flow и каталог MCP.

**Подвох.** Нужен Nix с flakes и home-manager; поддержка только aarch64-darwin и x86_64-linux; локальный MLX-инференс (macOS) простаивает, пока не настроишь. Порог входа высокий, не для новичков.

`mcp · automation · hooks · skills` · Источник: [github/dryvist](https://github.com/dryvist/nix-ai)

## 10. iris-agentic-dev: single-binary MCP-сервер, чтобы Claude Code работал с InterSystems IRIS

**Зачем.** Разработчикам на InterSystems IRIS/ObjectScript, кто хочет, чтобы Claude Code реально проверял и запускал код в IRIS, а не гадал.

**Как.** Поставить бинарник (`brew install intersystems-community/iris-agentic-dev/iris-agentic-dev` или curl из релизов) и прописать MCP-сервер в `~/.claude.json` с env IRIS_HOST/PORT/USERNAME/PASSWORD/NAMESPACE. Инструменты: iris_compile, iris_execute, iris_query, iris_test, iris_symbols, iris_coverage, iris_debug и др.

**Подвох.** Узкая ниша (нужен IRIS 2023.1+); на Windows требуется настроенное IIS `/api` (частая ошибка — 404 на /api/atelier); write-операции на проде гейтятся, пока не выставишь IRIS_ALLOW_PROD=1.

`mcp · testing` · Источник: [github/intersystems-community](https://github.com/intersystems-community/iris-agentic-dev)

## 11. Гайд: как отдать запасной Mac под управление Claude Code (SSH + computer use)

**Зачем.** Тем, кто боится пускать автономного агента на основной компьютер: изолировать его на отдельной машине.

**Как.** Включить SSH (`sudo systemsetup -setremotelogin on`), настроить passwordless sudo, отключить сон (`sudo pmset -c sleep 0`), поставить Claude Code (`curl -fsSL https://claude.ai/install.sh | bash`); для управления экраном — `setup-computer-use.sh` через SSH.

**Подвох.** macOS требует вручную выдать права Screen Recording и Accessibility (это не скриптуется); computer use нужен план Pro/Max; инструкция про Mac, не Windows/Linux.

`automation · tips` · Источник: [hn/ykdojo](https://ykdojo.github.io/claude-controls-mac/)
