## 2. Anthropic: крупные миграции кода флотом агентов Claude Code

**Зачем.** Командам перед большой миграцией (смена языка/рантайма, техдолг на масштабе) — рабочий шаблон, как это делают через флот агентов, а не вручную.

**Как.** Сначала пишется «rulebook» = спецификация для остальных агентов; правила обкатываются на нескольких файлах; очередь задач механическая («done = файл существует»); судья — компилятор и тесты, не человек. Sonnet — параллельная реализация, Fable 5 / Opus — состязательное ревью. Результаты: Bun Zig→Rust ~1 млн строк за <2 недель при 100% тестов; внутренний Python→TypeScript — 165k строк за выходные; ~5.9 млрд входных / 690 млн выходных токенов (~$165k).

**Подвох.** Масштаб и бюджет корпоративные. Общий урок: вложиться в правила и тест-судью ДО фан-аута, иначе агенты множат ошибки быстрее.

`refactoring · automation · subagents` · Источник: [blog/Anthropic](https://claude.com/blog/ai-code-migration)

## 3. Claude-Code-Agent-Monitor: real-time дашборд сессий и субагентов

**Зачем.** Тем, кто отлаживает сложные многоагентные прогоны: видеть в реальном времени, что делают агенты, какие инструменты вызывают, сколько токенов жгут и как связаны родитель-субагент.

**Как.** В свой проект: `npm run setup`, затем `npm start`. Ловит события через хук-систему Claude Code, пишет в SQLite; фронт React+WebSockets, Kanban-доски, транскрипты, графики. Есть Electron-приложение и Docker.

**Подвох.** Вызовы инструментов субагентами не эмитят хуки — их приходится дочитывать из JSONL пост-фактум. Инструмент self-hosted.

`subagents · hooks · tips` · Источник: [github/hoangsonww](https://github.com/hoangsonww/Claude-Code-Agent-Monitor)

## 6. claude-lane-stack: оркестрация параллельных агентов с авто-мержем

**Зачем.** Кто ведёт несколько агентов параллельно и устал от пяти чатов и ручных мержей — превратить хаос в конвейер с одним координатором.

**Как.** `git clone` + `./install.sh`, добавить `~/.agents/bin` в PATH; `agents-doctor --apply .`, запуск `claude --agent dev-orchestrator`. YAML-контракты задают `owns_paths` (лейны не конфликтуют); проверенное авто-мержится в `main`.

**Подвох.** Claude Code обязателен (Grok/Codex опциональны, есть режим `claude-only`); для Linux-Grok нужен `bubblewrap`. Не универсальный мульти-LLM.

`workflows · subagents · automation` · Источник: [github/VKirill](https://github.com/VKirill/claude-lane-stack)

## 8. Claude Code: анатомия «мисфичи» — тихий авто-continue агента

**Зачем.** Всем, кто запускает Claude Code без присмотра (CI, ночные прогоны, деплой): узнать про недокументированное поведение, обходящее намеренный барьер согласования.

**Как.** С 2.1.198 при отсутствии ответа на `AskUserQuestion` за 60с агент продолжает «на своё усмотрение» (отсчёт виден лишь последние 40с). Автор советует запинить CLI: в `~/.claude/settings.json` env `DISABLE_AUTOUPDATER=1` + `FORCE_AUTOUPDATE_PLUGINS=1`; обсуждается `CLAUDE_AFK_TIMEOUT_MS`.

**Подвох.** Фича приехала без changelog и доков — это критическая позиция автора. В safety-critical она превращает блокирующий шаг в 60-сек. байпас.

## 13. Как срезать расход токенов Claude Code на 70% одним мастер-промптом

**Зачем.** Кто хочет снизить расход токенов без потери качества — экономия от того, чтобы не объяснять одно и то же дважды.

**Как.** Единый «мастер-промпт»: роль/контекст, «do-not-touch»-список, «Phase 0 Discovery» через grep, нумерованные issues с before/after, чек-лист приёмки. Плюс привычки: «discovery before code», реальная верификация, «один коммит — одна тема».

**Подвох.** 70% — личная оценка без замеров; готовые шаблоны спрятаны в платном продукте автора.
