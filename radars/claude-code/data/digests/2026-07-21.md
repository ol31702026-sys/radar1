## 1. JetBrains разобрала RTK-хук: «экономия токенов» оказалась удорожанием

**Зачем.** Тем, кто ставит «токеносберегающие» хуки и обёртки на Claude Code и верит их встроенным счётчикам экономии — статья показывает, как такие метрики вводят в заблуждение.

**Как.** RTK — прокси, который PreToolUse-хуком переписывает вывод shell-команд (напр. `git status` в компактный вид). JetBrains прогнала парные замеры: медиана +7.6% стоимости на низком effort (p=0.004), ±0% на высоком. Причины: хук трогает ~20% вывода (Read/Grep идут мимо), Claude Code и так обрезает большие результаты, а основная входная цена — кешированные перечитывания по 1/10 цены.

**Подвох.** Это разбор одного инструмента на конкретных задачах. Но вывод «мерить оплаченный счёт, а не дифф инструмента» универсален.

`cost-tokens · tips` · Источник: [blog/JetBrains](https://blog.jetbrains.com/ai/2026/07/rtk-claude-code-token-savings/)

## 2. effort-router: авто-подбор /effort под каждый запрос

**Зачем.** Тем, кто вручную дёргает `/effort` или забывает его понизить: срезает токены, не проседая по качеству на сложном.

**Как.** Ставится как plugin. Хук `SessionStart` вкладывает классификатор, `UserPromptSubmit` переякоривает маршрут на каждый промпт (~20 токенов). Пять скиллов = пять тиров (low…max), оверрайд на один ход. Скилл `effort-report` считает сэкономленное, лог в `~/.claude/effort-router/usage/`.

**Подвох.** +1 круг модели на ход (~1–3 c); оверрайд держится один ход и не виден в UI; старые сессии не логируются, пока не перезапустишь.

`cost-tokens · hooks` · Источник: [github/cfitzgerald-pd](https://github.com/cfitzgerald-pd/effort-router)

## 3. agent-yes: гонять Claude Code без няньки

**Зачем.** Кто запускает долгие/ночные прогоны и устал жать подтверждения и ловить обрывы на лимитах.

**Как.** `curl -fsSL https://agent-yes.com/setup.sh | sh` (или `npm i -g agent-yes`), запуск `ay claude "задача"`. Сам отвечает на типовые prompt'ы, детектит простой, перезапускает упавшие процессы; управление живыми агентами — `cy ls`, `cy tail`, `cy send`.

**Подвох.** «Отвечает Yes без подтверждения» → запускать ТОЛЬКО на доверенных репозиториях, иначе агент сделает необратимое без спроса.

`automation · tips` · Источник: [github/snomiao](https://github.com/snomiao/agent-yes)

## 4. Synapse: локальный индекс кодовой базы как MCP-сервер

**Зачем.** На средних/крупных репозиториях — чтобы Claude сам подтягивал нужные файлы и соглашения проекта, а вы не тратили сообщения на «вот ещё этот файл».

**Как.** `pip install synapse-mcp`, `synapse init C:/Projects/myapp`, `claude mcp add synapse synapse-mcp`, перезапуск. Семантический поиск подмешивает сниппеты в контекст; всё локально. Нужен Python 3.10+, ~1 ГБ.

**Подвох.** v0.1.0 (19 июля), ранняя: чат Claude Desktop только в v0.2; по умолчанию индексирует и `.env` — вычистите секреты из области индексации.

`mcp` · Источник: [github/nrkoka786](https://github.com/nrkoka786/synapse)

## 5. amnesia: аудит памяти Claude Code на противоречия

**Зачем.** У кого накопилась память проекта: агент уверенно применяет устаревшие факты (старый порт, отменённый сервис) — и это тихо портит результат.

**Как.** `/plugin marketplace add tiny-cloud-ventures/amnesia`, `/plugin install amnesia@amnesia`, затем `/amnesia:open` (или CLI `amnesia` → веб-UI `http://localhost:8780`). Граф связей памяти, ревью в один клик, удаление через «корзину» (обратимо). Локально, через ваш же Claude CLI.

**Подвох.** Консолидацию всё равно подтверждаете руками; польза есть, только если вы реально ведёте память проекта.

`tips` · Источник: [github/tiny-cloud-ventures](https://github.com/tiny-cloud-ventures/amnesia)

## 6. OtoDock: команда агентов Claude Code на своём сервере

**Зачем.** Небольшой команде, которой нужен общий доступ к агентам Claude Code без раздачи ключей: единый сервер, роли, аудит, автоматизация по расписанию/вебхукам.

**Как.** Docker: `curl -fsSLO .../scripts/install.sh` и `bash install.sh`; дашборд на `http://localhost:8400`. Внутри: мультиагентный чат со стримингом инструментов, генерация Word/Excel/PPT/PDF, MCP-инструменты, sandbox с правами, SSO/OIDC, 2FA. Движок — «настоящие» Claude Code и Codex.

**Подвох.** v1.2.0, ранняя (≈13 звёзд); лицензия fair-source (FSL-1.1) ограничивает коммерческую конкуренцию; нужен Docker или голый Debian/Ubuntu.

`automation · subagents` · Источник: [github/OtoDock](https://github.com/OtoDock/oto-dock/)

## 7. aloud: голосовые уведомления для Claude Code

**Зачем.** При долгих прогонах не сидеть глазами в терминале: услышите, когда агент застрял на вопросе или завершил задачу.

**Как.** Homebrew + pipx, затем `aloud install`, `aloud doctor`, перезапуск Claude Code, права Hammerspoon Accessibility; в сессии `aloud on`. Команды `aloud full`, `aloud repeat`; горячие клавиши Cmd+Ctrl+H/J/. . Голос — локальный Kokoro.

**Подвох.** Только macOS, Python 3.11–3.12, синтез на CPU; первый запуск качает модель с Hugging Face; редакция секретов не идеальна — проверяйте, что озвучивается.

`tips · hooks` · Источник: [github/softcane](https://github.com/softcane/aloud)

## 8. pexels-skill: скилл Claude Code для поиска бесплатных стоковых фото

**Зачем.** Фрилансеру/малому бизнесу при сборке сайта или контента: не отвлекаться на ручной поиск стоковых фото, дать Claude искать их по запросу.

**Как.** `git clone .../pexels-skill.git ~/.claude/skills/pexels`, `export PEXELS_API_KEY=...`, вызов `bash ~/.claude/skills/pexels/scripts/search.sh "surfing sunset beach" 10 landscape`.

**Подвох.** Нужен бесплатный ключ Pexels; это bash-скрипт (под Windows без bash не заведётся); репозиторий крошечный (1 коммит), без гарантий поддержки.

`skills` · Источник: [github/amalshehu](https://github.com/amalshehu/pexels-skill)
