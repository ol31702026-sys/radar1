## 1. Datadog собрал «универсальный станок» (Temper) для Claude Code

**Зачем.** Инженерным командам, где агенты уже пишут много кода, но люди всё ещё вручную координируют деплой — идея «детерминированного ядра», превращающего работу агента из генерации кода в производство спецификаций.

**Как.** Temper — детерминированное ядро: агент выдаёт спецификацию, ядро верифицирует её через четыре слоя анализа и деплоит. Путь к нему — три проекта: Courier (очереди, 2024), BitsEvolve (эволюционная оптимизация, 2025) и Helix (стриминг уровня Kafka, собран Claude Code почти автономно за несколько дней с одним человеком у руля).

**Подвох.** Конкретных метрик продуктивности в статье нет (сказано лишь, что Claude Code даёт «минимум две трети» AI-кодинга инженеров Datadog для прода); это архитектурный нарратив крупной компании, а не пошаговый гайд — «станок» заточен под их инфраструктуру.

`automation · workflows · case-study` · Источник: [blog/Datadog / Anthropic](https://claude.com/blog/how-datadog-built-a-universal-machine-tool-for-claude-code)

## 2. Как Anthropic держит безопасность, когда Claude пишет ~80% кода

**Зачем.** Командам, которые пускают агентов в прод-код и боятся, что «агент что-то сломает или утащит» — набор практик, которые можно перенять, чтобы обезопасить AI-разработку.

**Как.** На планировании — авто Project Security Review на Opus против MITRE ATT&CK; при генерации — правила безопасного кода в CLAUDE.md, команда /security-review ищет атакуемые входы и подозрительные ссылки, плагин security-guidance советует по ходу, удалённые VM с egress-allowlist ограничивают ущерб от инъекции; в CI — несколько узких ревью-агентов с RAG по прошлым инцидентам, PreToolUse-хуки для жёстких гейтов, риск-tiered одобрения (критичное — человеком); в проде — DAST в стейджинге, агенты с минимальными правами, логи в SIEM.

**Подвох.** Это масштаб Anthropic (агенты проверяют друг друга, отдельные identity, shadow-mode для новых ревьюеров) — маленькой команде столько слоёв не нужно; практичная выжимка для себя: правила в CLAUDE.md + /security-review + PreToolUse-хук на опасные действия.

`hooks · ci-cd · automation` · Источник: [blog/Anthropic](https://claude.com/blog/how-anthropic-secures-its-ai-native-software-development-lifecycle)

## 15. Автоматизация первой линии поддержки на Claude Code + MCP (кейс основателя)

**Зачем.** Соло-основателю или микрокоманде — снять рутину первой линии поддержки: рассортировать входящие, поднять контекст и подготовить ответы, оставив отправку за человеком.

**Как.** Claude Code запускается по расписанию (launchd/cron) с четырьмя MCP-серверами (тул поддержки, Sentry, Nightwatch, почта); markdown-файл задаёт правила, JSON-файл хранит состояние. Агент берёт треды с активностью с прошлого прогона, классифицирует (срочное/лид/шум), сверяет с прод-ошибками и кодом, проверяет почту на дубли, создаёт черновики (никогда не отправляет), архивирует и репортит. Первый прогон: 5 тредов → 3 черновика, 2 верно пропущены.

**Подвох.** Честно оговорено — это «масштаб основателя, не энтерпрайз-поддержка»: нет доступа к прод-данным аккаунтов, нужна жёсткая письменная установка «проверяй или не пиши», модель «только черновики» не разгружает большие объёмы.

`mcp · automation · case-study` · Источник: [hn/kizum](https://sitespeak.ai/blog/automate-customer-support-mcp)
