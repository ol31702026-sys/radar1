## 1. Claude Code v2.1.217: лимит на параллельные сабагенты и жёсткий стоп по бюджету

**Зачем.** Тем, кто гоняет параллельные сабагенты и боится «разлёта» задач и счёта за токены: появились предсказуемые тормоза.

**Как.** Потолок одновременных сабагентов — env `CLAUDE_CODE_MAX_CONCURRENT_SUBAGENTS` (по умолчанию 20); вложенные сабагенты по умолчанию отключены (глубину задаёт `CLAUDE_CODE_MAX_SUBAGENT_SPAWN_DEPTH`); при достижении `--max-budget-usd` новые запуски запрещаются, а работающие фоновые агенты останавливаются. Плюс автодополнение emoji-шорткодов и починка утечки памяти на обрезанных MCP-ответах.

**Подвох.** Это «дневная» сборка с быстрым циклом релизов: значительная часть — багфиксы под конкретные окружения (Windows-автообновление, корпоративные mTLS/proxy). Обновляйтесь, но перепроверьте свои env-переменные.

`subagents · cost-tokens · automation` · Источник: [github/anthropics](https://github.com/anthropics/claude-code/releases/tag/v2.1.217)

## 2. Claude Code v2.1.216: отдельный выключатель изоляции ФС и фикс квадратичного тормоза

**Зачем.** Тем, у кого песочница мешает работать с файлами, но нужен контроль egress; и всем, у кого длинные сессии начинали «залипать».

**Как.** Выставьте `sandbox.filesystem.disabled`, чтобы пропустить изоляцию ФС, но оставить сетевой контроль. Плюс исправлено нарастание стоимости нормализации сообщений (росло квадратично от числа ходов → многосекундные паузы и медленный resume) и улучшено сообщение об ошибке в `/ultrareview`.

**Подвох.** Отключение изоляции ФС снижает защиту — включайте осознанно. Релиз в основном про 40+ багфиксов (auth, парсинг Bash/PowerShell, worktree), а не про новые фичи.

`automation · workflows` · Источник: [github/anthropics](https://github.com/anthropics/claude-code/releases/tag/v2.1.216)

## 3. Claude Code v2.1.212: /fork уводит беседу в фоновую сессию, лимиты на сабагентов и поиск

**Зачем.** Тем, кто распараллеливает работу и хочет ветвить контекст без остановки основной сессии, и кому важна защита от бесконечных циклов делегирования/поиска.

**Как.** `/fork` копирует беседу в новую фоновую сессию (внутрисессионный сабагент заменён на `/subtask`); лимиты — `CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION` и `CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION` (оба 200, сброс — `/clear`). MCP-вызовы дольше 2 минут уходят в фон автоматически; `/resume` открывает пикер прошлых сессий.

**Подвох.** Поведение `/fork` изменилось (раньше запускал внутрисессионный сабагент) — если были свои скрипты или привычки, перепроверьте. «Дневные» релизы идут близко друг к другу — легко перепутать версии.

`subagents · slash-commands · automation` · Источник: [github/anthropics](https://github.com/anthropics/claude-code/releases/tag/v2.1.212)

## 4. Claude Security Plugin для Claude Code — бета: скан, проверка и патч уязвимостей внутри CLI

**Зачем.** Командам, которым нужен контекстный security-ревью (не только паттерн-матчинг) без отправки кода наружу.

**Как.** Claude сканирует репозиторий, валидирует находки и предлагает исправления на ревью; ищет high-severity: повреждение памяти, инъекции, обход аутентификации, сложные логические ошибки, которые часто пропускают линтеры. Работает как плагин внутри Claude Code; код остаётся в вашем окружении.

**Подвох.** Пока бета — доверяй, но проверяй патчи руками. Часть страницы посвящена энтерпрайз-продукту Claude Security; конкретики по установке плагина и лимитам на ней немного.

`testing · automation · tips` · Источник: [docs/Anthropic](https://claude.com/product/claude-security)

## 5. Fireside chat с командой Claude Code: меньше правил в промпте, больше контекста

**Зачем.** Всем, кто пишет CLAUDE.md и системные промпты для агентов: практические принципы из первых рук.

**Как.** Ключевое: примеры в промпте для продвинутых моделей больше не нужны (модель креативнее ваших примеров) — системный промпт CC сократили на ~80%; давайте контекст вместо списков «не делай»; почти все в Anthropic работают в auto mode (после тысяч evals риск prompt injection ниже, чем у среднего ревьюера-человека); «Claude Tag» закрывает ~65% product-инженерных PR и лучше работает в публичных каналах.

**Подвох.** Это опыт Anthropic на их процессах и моделях — не догма для вашего стека; auto mode всё же требует осознанного отношения к безопасности.

`prompting · workflows · tips` · Источник: [blog/Simon Willison](https://simonwillison.net/2026/Jul/21/cat-and-thariq/)

## 7. Как гонять несколько сессий Claude Code параллельно и не ловить конфликты

**Зачем.** Тем, кто запускает 2+ сессии Claude Code одновременно и упирается в конфликты файлов и веток.

**Как.** Шесть приёмов: команды continue/resume и recap/export для переноса контекста; правила владения файлами (одна сессия пишет, другие — read-only); Plan Mode делит задачу на независимые части; git worktree — отдельная ветка и директория на сессию; оркестрация через форк OpenAI Symphony по GitHub-issue; Docker-контейнеры (Clone Mode) для полной изоляции.

**Подвох.** Это обзорный гайд, а не пошаговый рецепт под ваш стек; часть инструментов (форк Symphony) — сторонние, проверяйте их актуальность.

`workflows · subagents · automation · tips` · Источник: [blog/Frank La Vigne](https://www.franksworld.com/2026/07/21/mastering-claude-code-a-guide-to-effective-parallel-ai-workflow/)

## 8. UiPath/coder_eval: воспроизводимые YAML-бенчмарки для Claude Code с CI-гейтами

**Зачем.** Тем, кто пишет скиллы и агентов и хочет ловить регрессии, когда скилл «тихо» перестаёт срабатывать при дрейфе моделей.

**Как.** Задачи — декларативные YAML (промпт, конфиг агента, песочница, критерии успеха); оценка 0.0–1.0 по критериям (наличие файла, выполнение команды, схожесть кода, LLM-рубрики), есть проверка `skill_triggered`. Установка: `uv tool install coder-eval`; запуск: `uv run coder-eval run tasks/hello_date.yaml`. CI-гейты валят билд на регрессиях.

**Подвох.** Требует Python 3.13+, uv 0.8+ и Claude CLI; проект молодой (~105★), Apache-2.0 — заложите время на написание YAML-сьютов.

`testing · ci-cd · skills` · Источник: [github/UiPath](https://github.com/UiPath/coder_eval)

## 9. Pull-based JSON-леджер для передачи задач между сессиями Claude и Codex

**Зачем.** Тем, кто связывает две несинхронные агент-сессии (одна регистрирует задачу и завершается, другая позже её подхватывает) без внешней шины сообщений.

**Как.** Каждая запись: `id`, `owner`, `requester`, `stage`, `status` (running/complete), `next_check_at` (когда опрашивать снова), `max_silence_minutes` (дедлайн). Три команды: `start` (регистрирует задачу), `check` (опрашивает; exit code 3, если просрочено), `complete` (закрывает). Общее состояние живёт в репозитории, аудит — через git-историю.

**Подвох.** Это опрос, а не реалтайм — задержки зависят от `next_check_at` и расписания Actions; подходит для нечастых хендоффов, не для тесной синхронизации.

`workflows · ci-cd · automation · tips` · Источник: [blog/morinaga](https://dev.to/morinaga/how-i-coordinate-claude-and-codex-sessions-with-a-pull-based-json-handoff-ledger-5h93)

## 10. razzant/claudexor: маршрутизация по квотам между Claude Code, Codex и Cursor

**Зачем.** Тем, у кого несколько подписок Claude/Codex/Cursor и кто упирается в лимиты или не видит реальной стоимости прогонов.

**Как.** Установка: `npm install -g claudexor`, затем `claudexor doctor`, `claudexor ask "..."`, `claudexor agent "fix the failing test" --harness codex`. Показывает проценты за 5 часов / 7 дней / по моделям; ротирует «выжатый» аккаунт (по явным лимитам вендора); умеет best-of-N с независимыми ревьюерами и арбитражем.

**Подвох.** Требует Node ≥20.19 и pnpm; десктоп-приложение только под macOS (CLI работает на Linux); работа с несколькими аккаунтами — вопрос к ToS вендоров, применяйте на свой риск.

`cost-tokens · automation · workflows` · Источник: [github/razzant](https://github.com/razzant/claudexor)

## 12. eigenwise-toolshed: плагины Claude Code с самообновляемой картой кодовой базы

**Зачем.** Тем, кто устал каждый раз объяснять Claude структуру проекта — карта архитектуры сохраняется между сессиями.

**Как.** Установка: `/plugin marketplace add Eigenwise/eigenwise-toolshed`, затем `/plugin install codebase-mapper@eigenwise-toolshed` (и по желанию `workbench`, `live-rules`, `sidequest`), `/reload-plugins`. codebase-mapper создаёт атомарные Markdown-документы в `.claude/.codebase-info/` и переинъектит карту на каждый запрос; работает на любом стеке.

**Подвох.** Проект небольшой (~10★), хоть и активный (889 коммитов); постоянная переинъекция карты добавляет контекст — следите за расходом токенов на больших репозиториях.

`workflows · automation · tips` · Источник: [github/Eigenwise](https://github.com/Eigenwise/eigenwise-toolshed)

## 13. zetetic-team-subagents: сабагенты «докажи источником» и пре-коммит-гейт против выдуманных чисел

**Зачем.** Тем, кого раздражает «уверенно, но неверно»: система делает признание незнания фичей, а не провалом.

**Как.** Установка: `claude plugin marketplace add cdeust/zetetic-team-subagents`, затем `claude plugin install zetetic-team-subagents`; только гейты — `claude plugin install zetetic-gates` (30 сек). 11 скиллов и 97 условий отказа; пре-коммит помечает keywords вроде «always/never/obviously» и float-константы с 3+ знаками без аннотации `source:`.

**Подвох.** Хук проверяет наличие ссылки, а не её истинность; срабатывает только в пути вызова Claude Code (прямой git-commit из терминала обходит гейт); целочисленные константы не флагуются. Проект нишевый (~7★).

## 15. Кейс: Claude Code собрал прошивку для ESP32 и нашёл баг UI, «посмотрев» в веб-камеру

**Зачем.** Инди-разработчикам и хардварщикам как пример того, что агент может закрывать цикл «код → железо → визуальная проверка» почти без ручного участия.

**Как.** Скилл отдаёт Claude кадры с камеры как входные данные: он сам снимает экран устройства, интерпретирует вывод, правит код, перепрошивает и перепроверяет — без ручного цикла «скриншот → вставил в чат». Так был найден баг: шрифт с недостаточным покрытием глифов «съедал» символы на панели, а тесты этого не видели. Установка ~2 минуты, MIT, v1.0.0.

**Подвох.** Это проект-демо (~21★); нужен источник видеопотока (Android IP Webcam / Raspberry Pi / HTTP-снимки); визуальная проверка агентом полезна, но не заменяет нормальные тесты.

`case-study · testing · tips` · Источник: [github/fcavalcantirj](https://github.com/fcavalcantirj/claude-code-eyes)
