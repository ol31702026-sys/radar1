## О чём видео

Это полный гайд по построению **agent loops** (циклов агентов) и автономных AI-систем. Вокруг
термина «loop engineering» много хайпа, но мало кто показывает, как применять эти идеи на
практике. Многие мечтают об автономных агентах, работающих 24/7 — делающих работу, зарабатывающих,
экономящих время. Но в реальности построение агентных систем требует **настоящего проектного
мышления (design thinking)**.

Автор (Owen — инженер, инженерный менеджер) показывает, как он использует **связку Claude Code
+ Codex**, чтобы автоматизировать значительную часть процесса разработки.

## Суть подхода (agent loop)

Цикл агента — это повторяющаяся петля, в которой агент:
1. **планирует** следующий шаг,
2. **выполняет** действие (правка кода, запуск команды),
3. **проверяет** результат (тесты, наблюдение за поведением),
4. на основе проверки решает, что делать дальше — и идёт на новую итерацию.

Ключевая мысль: автономность не берётся сама — её нужно **спроектировать**. Где гейты
человеческого контроля, что агент проверяет сам, как он понимает «готово», какую часть отдать
Claude Code, а какую Codex.

## Связка Claude Code + Codex

Автор комбинирует два инструмента, распределяя роли между ними, чтобы покрыть больше этапов
разработки автономно. Все промпты и ресурсы он выкладывает бесплатно в описании к видео.

## Вывод

Хороший практический ввод в проектирование автономных пайплайнов: не «магия 24/7», а
продуманная петля план → действие → проверка → решение, с явными точками контроля.
