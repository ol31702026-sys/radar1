# Codex собирает iOS-приложения — угонится ли Claude Code?

> Источник: https://www.youtube.com/shorts/zLHkvWMLEHI
> Тип: транскрипция видео (русский перевод субтитров)
> Канал: Hyperautomation Labs

_Полный перевод транскрипции (Shorts)._

OpenAI только что дал Codex «глаза» внутри симулятора iPhone. И я задал очевидный вопрос:
**а может ли так Claude Code?**

Новый плагин Codex «build iOS apps» пишет SwiftUI, запускает его в симуляторе, нажимает
кнопки, делает скриншоты того, что видит, и сам чинит свои баги. Агент наконец-то наблюдает
за тем, как работает его собственное приложение. Кажется, будто OpenAI изобрёл «замыкание
цикла» (closing the loop). Но нет.

Под этим плагином «в один клик» лежит **Xcode Build MCP** — открытый инструмент, который может
подключить кто угодно. А значит, **Claude Code крутит тот же цикл**: собрать → запустить →
скриншот → самоотладка. С одним установленным MCP, без фирменного плагина — тот же замкнутый
цикл.

И то, о чём никто не упоминает: Apple уже встроила Claude прямо в **Xcode 26.3**. Живые
превью SwiftUI, агент наблюдает за перерисовкой канваса прямо по мере набора кода.

Так кто победил? Codex красиво упаковал это. А Claude Code делает то же самое через **открытые
стандарты** (MCP).
