## Что такое субагенты (и чего о них чаще всего не понимают)

Вы просите Claude Code добавить фичу, и на полпути он тратит тысячу токенов на grep по
кодовой базе, а затем сваливает результаты в тот же диалог, который вы пытаетесь держать
чистым. Субагент решает ровно эту проблему. Он делает «грязную» побочную работу в своём
собственном окне контекста и возвращает только сводку — так ваш основной поток остаётся
сфокусирован на реальной задаче.

Это полный гайд по субагентам Claude Code: что это такое (и в чём большинство статей
ошибаются насчёт параллелизма), полная схема frontmatter поле за полем, как Claude решает
делегировать, передний план против фона, выстраивание ревьюера в цепочку с исправителем и
сообществные библиотеки, которые стоит «украсть». Все команды (включая агента, который
ниже ловит настоящую SQL-инъекцию) выполнялись на живой установке.

Всё это запускалось на Claude Code 2.1.177 в июне 2026.

## Ключевая идея

Субагент — это специализированный помощник со своим системным промптом, своим набором
доступных инструментов и своими правами (permissions). Он выполняет ограниченную задачу
(исследование, ревью, прогон тестов, проверку безопасности) в изолированном контексте и
возвращает главному агенту только итог. Главный агент при этом владеет планированием и
интеграцией результатов.

Важная оговорка про параллелизм, которую многие упускают: субагенты хороши, когда задачи
независимы и исполнителям НЕ нужно общаться между собой. Если нужна координация на лету —
это уже другой механизм (агентные команды).
