# 7 облачных трюков Claude Code — агенты, что работают, пока ты спишь

> Источник: https://www.youtube.com/watch?v=lMdsMiPX6iw
> Тип: транскрипция видео (русский перевод субтитров, связный конспект)
> Канал: Hyperautomation Labs

_Перевод ключевого содержания транскрипции. Видео длинное — изложена суть по разделам._

## Главная идея

Почти все запускают Claude Code одинаково — в терминале, на своём ноутбуке, с открытой
крышкой, нянча каждый шаг. Но есть **целый второй слой**, которого большинство не видит: слой,
где агент продолжает работать **после того, как ты закрыл крышку**.

Представь: ты спишь, ноутбук выключен, а на серверах Anthropic агент с твоим именем уже открыл
pull request, прогнал тесты и оставил тебе сводку к утру. Это не фантастика — это работает уже
сейчас, просто почти никто не пользуется, потому что бо́льшая часть этого **не в меню, на
которое кликают**.

## Честная оговорка

Локальный Claude Code на твоей машине и облачная версия — **два разных зверя**. Облачный слой
новее, и часть возможностей всё ещё помечена как research preview (значит, лимиты могут
меняться). В видео автор отмечает, какие именно.

## Карта семи трюков (о чём речь)

1. **Автономные агенты по расписанию** — работают, пока компьютер выключен.
2. **Полноценная облачная машина**, которой ты управляешь со своего устройства.
3. Запуск задач, которые продолжаются часами/днями, с сохранением прогресса.
4. Агент сам открывает PR, гоняет тесты и оставляет сводку.
5–7. Прочие приёмы из «доков, которые никто не читает» — про управление облачными сессиями,
   подхват незавершённой работы и безопасные лимиты.

## Почему это важно для Radar

Этот трюк (агенты по расписанию в облаке, пока комп выключен) — ровно то, на чём строится
**Этап 2 нашего проекта** (ежедневный сбор по cron). Видео — хороший практический ввод в то,
как Anthropic-овские облачные/фоновые агенты устроены и где их пределы.
