## Если воркфлоу не работает — обычно дело в паттерне

Большинство разработчиков упираются в одну стену: Claude Code запущен, простые задачи решает
хорошо, а на чём-то посложнее ломается — задачи теряются, контекст переполняется, вывод
непоследователен. Виновата почти никогда не сама модель, а **неправильно выбранный паттерн
воркфлоу** под задачу.

## Пять агентных паттернов

1. **Sequential flow (последовательный)** — шаги один за другим. Для линейных задач.
2. **Operator (оператор)** — один управляющий агент дирижирует выполнением.
3. **Split-and-merge (разделить и слить)** — задача дробится на части, выполняется параллельно,
   результаты сводятся.
4. **Agent teams (команды агентов)** — несколько специализированных агентов координируются
   (когда им нужно обмениваться находками на лету).
5. **Headless (без интерфейса)** — запуск в скриптах/CI без интерактива.

## Главный принцип

Подбирай **структуру под форму задачи**: начинай проще, чем кажется нужным, и добавляй
сложность (план-ревью, субагентов, авто-режим) только когда более простой подход реально не
справляется. Не «модель плохая» — а «паттерн не тот».
